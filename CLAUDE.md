# telegramassist — Claude Code Context

## Что за проект и для кого
Персональный Telegram-бот ассистент для одного пользователя. Работает 24/7 на VPS, помнит всё между сессиями. Отслеживает прогресс в вайбкодинге (streak, темы, понимание) и личные цели (тренировки, достижения).

## Стек
- **Язык:** Python
- **Telegram:** python-telegram-bot
- **LLM:** Groq API (Llama 3.3 70B, бесплатный тир)
- **MCP:** FastMCP (Python)
- **Память:** Markdown файлы (метод Карпатого — три уровня)
- **Логи:** SQLite
- **Деплой:** Docker Compose на VPS

## Архитектура памяти (метод Карпатого, 3 уровня)
```
memory/
├── raw/          ← уровень 1: хаотичный ввод, всё подряд из чатов
├── compiled/     ← уровень 2: агент структурирует raw по темам
│   ├── core.md
│   ├── learning.md
│   └── goals.md
└── wiki/         ← уровень 3: индекс — где что лежит, быстрый поиск
    └── index.md
```
Поток: raw → (агент компиляции) → compiled → wiki обновляется автоматически.

## Структура проекта
```
src/
├── bot/          ← Telegram-бот, slash-команды
└── mcp/          ← FastMCP сервер, инструменты памяти
memory/           ← три уровня памяти (см. выше)
docs/             ← документация проекта
docker-compose.yml
```

## Компоненты
- **telegram-bot** — принимает сообщения, читает память, вызывает Groq, отвечает
- **mcp-server** — инструменты: read_memory, write_memory, search_memory, web_search, log_conversation
- **Slash-команды:** /progress /remember /plan /streak /forget

## Решения, которые уже приняты
- Память хранится в markdown (не в БД) — для человекочитаемости и ручного редактирования
- Три уровня памяти: RAW → Compiled → Wiki (метод Карпатого)
- LLM — Groq (Llama 3.3 70B), не OpenAI — бесплатный тир достаточен
- Деплой через Docker Compose, не serverless — нужна постоянная работа 24/7
- MVP без голоса, веб-интерфейса и мульти-пользователей
