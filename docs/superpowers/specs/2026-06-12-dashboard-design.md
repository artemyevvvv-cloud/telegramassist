# Dashboard — Дизайн

**Дата:** 2026-06-12  
**Статус:** Approved

## Что это

Telegram Mini App — персональный дашборд прогресса. Открывается кнопкой прямо в боте. Показывает три направления: учёба, тренировки, работа. Акцент на учёбу.

## Платформа и хостинг

- **Frontend:** Vanilla HTML + CSS + JS, три файла (`index.html`, `style.css`, `app.js`)
- **Хостинг:** Vercel (бесплатно, статика)
- **Данные:** markdown-файлы памяти в GitHub-репозитории, читаются через `raw.githubusercontent.com`
- **Обновление данных:** `git push` — вручную или через кнопку в боте

Нет сервера, нет базы данных, нет платных сервисов.

## Визуальный стиль

- Тёмный фон: `#0d0d0d`
- Карточки с тонкой светящейся рамкой
- Акцентный цвет: синий/фиолетовый неон
- Шрифт: моноширинный (терминальный)
- Никаких лишних элементов, строгий минимализм

## Главный экран

Три карточки в ряд. Каждая показывает:
- Иконка + название блока
- Ключевая метрика (стрик / % выполнения / кол-во задач)
- Прогресс-бар
- Краткий статус

```
┌─────────────────┐  ┌─────────────────┐  ┌─────────────────┐
│  📚 УЧЁБА        │  │  💪 ТРЕНИРОВКИ   │  │  💼 РАБОТА       │
│  🔥 0 дней      │  │  4/5 на неделе  │  │  3 задачи       │
│  ████████░░ 70% │  │  ████░░░░░░ 40% │  │  ██████░░░░ 60% │

└─────────────────┘  └─────────────────┘  └─────────────────┘
```

Клик по карточке — раскрывается детальный вид.

## Детальный вид карточек

### 📚 Учёба
Источник: `memory/compiled/learning.md`

- Стрик: N дней подряд
- Прогресс: X из 9 модулей завершены
- Текущий модуль
- Список всех модулей с уроками: ✅ завершён / 🔶 в процессе / ⬜ не начат

### 💪 Тренировки
Источник: `memory/compiled/goals.md`

- % выполнения плана на текущей неделе
- Дни недели: закрашенные (выполнено) / пустые
- План от тренера на неделю
- По каждой тренировке: выполнил ✅/❌ + самочувствие (1–5) + усилие (1–5)

### 💼 Работа
Источник: `memory/compiled/goals.md`

- Кол-во активных задач
- Кол-во выполненных целей
- Список задач с галочками
- Выполненные цели

## Структура файлов goals.md

Для корректного парсинга дашбордом секции тренировок и работы в `goals.md` должны следовать этой структуре:

```markdown
## Тренировки

### Неделя 2026-06-09

| День | Упражнение | Выполнено | Самочувствие | Усилие |
|------|-----------|-----------|--------------|--------|
| Пн   | ...       | ✅        | 4            | 5      |

### План от тренера
- Пн: ...
- Вт: ...

## Работа

### Задачи
- [ ] Задача 1
- [x] Задача 2

### Цели
- [x] Цель 1
```

## Кнопка «Обновить дашборд» в боте

Новая кнопка в меню бота: **🚀 Обновить дашборд**

Логика:
1. Бот выполняет `git add memory/ && git commit -m "update memory [auto]" && git push`
2. Успех → отвечает "Дашборд обновлён ✅"
3. Ошибка → отвечает с текстом ошибки

Требует: переменная окружения `GITHUB_TOKEN` + настроенный remote с токеном в URL.

## Подключение Mini App к боту

Telegram Mini App подключается через BotFather:
1. `/mybots` → выбрать бота → Menu Button → настроить URL дашборда на Vercel
2. Кнопка «☰ Меню» в Telegram открывает дашборд

В боте дополнительно добавляется кнопка в reply-keyboard: **📊 Дашборд** — открывает тот же URL.

## Что не входит в MVP

- Графики и charts (линейный прогресс и числа достаточно)
- Уведомления из дашборда
- Редактирование данных прямо в дашборде
- Авторизация (дашборд персональный, публичный URL не страшен)
