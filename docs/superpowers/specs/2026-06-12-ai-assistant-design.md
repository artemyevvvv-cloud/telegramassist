# Личный ИИ-ассистент — Дизайн системы

**Дата:** 2026-06-12  
**Статус:** На согласовании

---

## Цель

Персональный ИИ-ассистент в Telegram, который работает 24/7 (даже когда компьютер выключен), помнит всё между сессиями и помогает отслеживать прогресс в обучении вайбкодингу и личные цели.

---

## Что отслеживаем

**Учёба (приоритет):**
- Пройденные темы по вайбкодингу
- Субъективная оценка понимания (1–5)
- Streak — сколько дней подряд занимаешься
- Блокеры и вопросы которые возникли

**Личное (легковесно):**
- Тренировки (тип, факт vs план)
- Достижения по работе
- Текущие активные цели

---

## Архитектура

```
VPS (сервер, работает 24/7)
└── docker compose
    ├── telegram-bot     ← принимает сообщения, общается с Groq
    ├── mcp-server       ← инструменты памяти и веб-поиска
    └── volume/
        ├── memory/      ← markdown файлы (метод Карпатого)
        └── logs.db      ← SQLite, история разговоров
```

**Интерфейсы:**
- Telegram — основной, с телефона в любой момент
- Claude Code — через MCP, когда работаешь за компом

---

## Память (метод Карпатого)

Простые markdown файлы. Человекочитаемые, редактируемые вручную.

```
memory/
├── core.md          ← кто ты, предпочтения, постоянные факты
├── learning.md      ← прогресс по вайбкодингу, пройденные темы
├── goals.md         ← спорт, работа, текущие цели
└── log/
    ├── 2026-06-12.md
    └── 2026-06-11.md
```

**Как работает:**  
Перед каждым ответом бот читает `core.md` + `learning.md` + последние 3 дня из `log/` и вставляет в системный промпт. После разговора извлекает новые факты и дописывает в нужный файл.

## Языковая модель

**Groq + Llama 3.3 70B**
- Бесплатный тир (~14K запросов/день)
- Быстрый ответ
- Достаточно умный для личного ассистента

## Компоненты

### telegram-bot
- Принимает сообщения от пользователя
- Определяет тип запроса (slash-команда / обычное сообщение)
- Обращается к mcp-server за памятью
- Отправляет запрос в Groq с контекстом
- Возвращает ответ

### mcp-server
Инструменты (tools):
- `read_memory(files)` — читает нужные файлы памяти
- `write_memory(file, content)` — записывает/дополняет файл
- `search_memory(query)` — ищет по всем файлам памяти
- `web_search(query)` — поиск в интернете
- `log_conversation(message, response)` — пишет в SQLite

### Slash-команды (Telegram)
- `/progress` — текущий прогресс по вайбкодингу
- `/remember <факт>` — явно сохранить что-то в память
- `/plan` — составить план на день
- `/streak` — показать streak по учёбе
- `/forget <факт>` — удалить устаревшее из памяти

## Ежедневный ритуал

**Утром** (бот пишет первым):
> "Доброе утро! Вчера ты изучил X. Что планируешь сегодня?"

**Вечером:**
> "Как прошёл день? Что удалось сделать по учёбе?"

После вечернего разговора бот автоматически обновляет `log/YYYY-MM-DD.md`.

**В любой момент:**
Пишешь боту — он отвечает с полным контекстом о тебе.

---

## Hooks (Claude Code)

- **Start hook** — при открытии сессии подгружает `core.md` и `learning.md` в контекст
- **Stop hook** — при завершении сессии сохраняет итоги в лог

## Docker Compose

Три сервиса:

| Сервис | Роль |
|--------|------|
| `telegram-bot` | Бот, слушает Telegram 24/7 |
| `mcp-server` | Память + поиск, порт 8000 |
| `volume` | Папка с memory/ и logs.db |

Деплой на VPS: `git pull && docker compose up -d`
