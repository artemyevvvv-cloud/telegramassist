# project-setup

Ты помогаешь человеку запустить новый проект — не просто создаёшь папки, а строишь полноценную рабочую зону: структуру управления, документацию, автоматическую память из сессий и хуки автоматизации.

## Шаг 1 — Задай все вопросы сразу

Задай следующие вопросы одним сообщением — коротко, по пунктам:

1. **Название** проекта
2. **Тип** — курс / SaaS / Telegram-бот / лендинг / AI-агент / другое (уточни)
3. **Для кого** — кто будет пользоваться / целевая аудитория
4. **Стек** — основные инструменты и технологии
5. **Платформа** — куда деплоить или публиковать
6. **GitHub** — уже есть репозиторий? (да + ссылка / нет)

Подожди ответа. Если после ответа не хватает информации для создания структуры — задай уточняющие вопросы. Если всего достаточно — сразу переходи к шагу 2.

## Шаг 2 — Создай структуру файлов

Создай все файлы, заполненные **реальным содержимым** на основе ответов пользователя. Не болванки, не «TODO», а конкретный текст для этого проекта.

### docs/brief.md
Суть проекта в 3-4 предложениях. Уникальная позиция. Для кого. Что создаётся на выходе.

### docs/status.md
Таблица с модулями/фичами/этапами (зависит от типа проекта). Статус каждого: ✅ готово / 🔶 в процессе / ❌ не начато. Начальный статус — всё ❌.

### docs/tasks.md
Топ-5 первых шагов, сгруппированных по приоритету. Конкретные действия, не абстракции.

### docs/issues.md
Пустой файл с заголовком и пояснением что сюда записывать.

### docs/ideas.md
Пустой файл с заголовком.

### docs/assets.md
Таблица: инструмент / ссылка / для чего используется. Заполни теми инструментами, которые назвал пользователь.

### CLAUDE.md
На русском, до 80 строк. Разделы:
- Что за проект и для кого
- Стек
- Структура проекта (для курса — модули, для продукта — фичи/компоненты)
- Текущий статус
- Карта файлов проекта
- Топ-3 приоритета сейчас
- Решения, которые уже приняты и не пересматриваются

### README.md
3-5 предложений: что это, для кого, как запустить (если применимо).

### .gitignore
Стандартный для типа проекта. Для любого проекта добавь:
```
archive/
tools/claude-memory-compiler/tools/
tools/claude-memory-compiler/reports/
tools/claude-memory-compiler/scripts/state.json
tools/claude-memory-compiler/scripts/last-flush.json
tools/claude-memory-compiler/scripts/*.log
tools/claude-memory-compiler/daily/
.env
*.env.local
```

## Шаг 3 — Настрой трёхуровневую память

Выполни последовательно:

**3.1** Клонируй репозиторий:
```bash
git clone https://github.com/coleam00/claude-memory-compiler tools/claude-memory-compiler
```

**3.2** Установи зависимости:
```bash
cd tools/claude-memory-compiler && uv sync
```

**3.3** Создай `.claude/settings.json` в корне проекта.

Возьми абсолютный путь к текущей рабочей директории (через `pwd` или `$PWD`) и подставь в команды хуков:

```json
{
  "hooks": {
    "SessionStart": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "cd \"<АБСОЛЮТНЫЙ_ПУТЬ>/tools/claude-memory-compiler\" && uv run python hooks/session-start.py",
            "timeout": 15
          }
        ]
      }
    ],
    "PreCompact": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "cd \"<АБСОЛЮТНЫЙ_ПУТЬ>/tools/claude-memory-compiler\" && uv run python hooks/pre-compact.py",
            "timeout": 10
          }
        ]
      }
    ],
    "SessionEnd": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "cd \"<АБСОЛЮТНЫЙ_ПУТЬ>/tools/claude-memory-compiler\" && uv run python hooks/session-end.py",
            "timeout": 10
          }
        ]
      }
    ]
  }
}
```

**3.4** Создай начальные файлы базы знаний:

`tools/claude-memory-compiler/knowledge/index.md`:
```markdown
# Knowledge Base Index

| Article | Summary | Compiled From | Updated |
|---------|---------|---------------|---------|
```

`tools/claude-memory-compiler/knowledge/log.md`:
```markdown
# Build Log

```

Создай папки: `tools/claude-memory-compiler/knowledge/concepts/`, `knowledge/connections/`, `knowledge/qa/`, `daily/`, `reports/`

**3.5** Проверь что хук работает:
```bash
cd tools/claude-memory-compiler && uv run python hooks/session-start.py
```
Должен вернуть JSON с `hookSpecificOutput`. Если ошибка — устрани.

## Шаг 4 — Итоговый отчёт

После завершения покажи:

```
✅ Рабочая зона создана

📁 Структура:
<дерево папок>

🧠 Трёхуровневая память:
— SessionStart: инжектирует базу знаний в каждую сессию
— SessionEnd: сохраняет разговор → daily/YYYY-MM-DD.md
— PreCompact: страховка перед сжатием контекста

🚀 Три первых шага:
1. <конкретный первый шаг для этого проекта>
2. <конкретный второй шаг>
3. <конкретный третий шаг>
```

Первые шаги должны быть конкретными для проекта пользователя, не общими фразами.
