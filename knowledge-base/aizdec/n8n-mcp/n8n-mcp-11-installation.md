## **[ ЧТО ТАКОЕ MCP-СЕРВЕР ]**

MCP-сервер это, можно сказать, программа, которая даёт Claude дополнительные знания и инструменты. А конкретно N8N-MCP это MCP-сервер для N8N. После их соединения вместе, мы получим агента который знает:

- _Все настройки N8N в деталях_

- _Тысячи готовых шаблонов_

- _Правила работы с ошибками_

**Важно понимать:** MCP-сервер запускается вместе с Claude Code. Выключаешь VS Code — он тоже выключается. Включаешь — запускается автоматически.

## **[ СПОСОБ 1: ЧЕРЕЗ АГЕНТА ]**

_Самый простой способ. Открой ч��т Claude Code и напиши:_

```text
Установи мне N8N-MCP в режиме user (для всех папок) из репозитория https://github.com/czlonkowski/n8n-mcp
```

**Всё. Агент сам разберётся что нужно:**

- Проверит есть ли Node.js (нужен для работы)

- Если нет — предложит установить

- Выполнит нужные команды

- Подключит MCP-сервер

Ты просто подтверждаешь когда он спрашивает.

💡 **Что происходит под капотом:
**Агент выполняет команду `claude mcp add n8n-mcp -s user -- npx n8n-mcp`. Вот что означает каждая часть:

| --- | --- |
|  |  |
|  |  |
|  |  |
|  |  |
|  |  |
|  |  |

⚠️ **Если MCP не работает в другом проекте:
**Скорее всего он установился в режиме local (только для одной папки). Попроси агента: "Переустанови N8N-MCP в режиме user для всех папок".

## **[ СПОСОБ 2: ЛОКАЛЬНАЯ ПАПКА ]**

_Альтернативный вариант — скачать N8N-MCP к себе на компьютер._

**Зачем это нужно:**

- Полный контроль над версией

- Можно модифицировать под себя

- Работает офлайн после первой загрузки

**Вариант A — через агента:**

Открой любую папку в VS Code и напиши агенту:

```text
Склонируй репозиторий https://github.com/czlonkowski/n8n-mcp в папку ~/Projects/n8n-mcp
```

Агент сам выполнит нужные команды. Ты только подтверждаешь.

**Вариант B — через интерфейс VS Code:**

1. Нажми **Cmd+Shift+P** (Mac) или **Ctrl+Shift+P** (Windows)

1. Напиши: **Git: Clone**

1. Вставь ссылку: `https://github.com/czlonkowski/n8n-mcp`

1. Выбери папку куда сохранить

1. VS Code скачает и предложит открыть

**После скачивания****:**

1. Открой папку n8n-mcp в VS Code

1. Claude увидит файл `.claude.md` с инструкциями и установит себя сам

1. После установки замени текст `.claude.md` на актуальный из README репозитория

⚠️ **Для большинства — способ 1 проще:
**Локальная папка нужна если хочешь глубже копаться в настройках. Для старта хватит установки через агента.

## **[ ПЕРЕЗАПУСК ]**

После установки нужно перезагрузить VS Code:

1. **Cmd+Shift+P** (Mac) или **Ctrl+Shift+P** (Windows)

1. Напиши: **Developer: Reload Window**

1. Enter

Или просто попроси агента: "Перезагрузи окно".

⚠️ **Первый запуск долгий:
**При первом запуске качается ~280MB данных. Это нормально. Следующие запуски мгновенные.

## **[ ПРОВЕРКА ]**

Не команда, а вопрос. Напиши агенту:

```text
Проверь, есть ли у тебя доступ к N8N-MCP и что ты видишь?
```

**Если работает** — ответит что-то типа: "Да, вижу 543 ноды, 2700 шаблонов, могу искать и проверять workflow".

**Если не работает** — скажет что не знает про N8N-MCP или ответит общими словами.
