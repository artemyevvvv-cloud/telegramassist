# Папка проекта

**Курс:** N8N + Claude Code  
**URL:** https://www.aizdec.me/courses/n8n-mcp/n8n-mcp-03-first-project

---

Claude Code видит только ту папку, которую ты открыл. Открыл папку с проектом — он видит все файлы внутри. Не открыл — ничего не видит.

Это как рабочий стол. Разложил документы — работаешь. Убрал в ящик — их нет.

## **[ ЗАЧЕМ НУЖНА ПАПКА ]**

В 📁 папке живёт всё, что относится к проекту:

- Экспортированные автоматизации из N8N

- Промпты для AI-агентов

- Заметки и документация

- Схемы, планы, TODO

_Claude читает эти файлы и понимает контекст. Спрашиваешь "что не так с моей автоматизацией?" — он открывает файлы, внимательно их изучает, и уже потом объясняет._

## **[ КАК ОРГАНИЗОВАТЬ ПАПКУ ]**

Вот один из рабочих вариантов. Не идеальный, но вполне допустимый.

**Корень проекта** — главная папка. В ней несколько подпапок и пара файлов:

- _**docs/**__ _— документация по проекту

- _**archive/**__ _— старые файлы, которые щас не нужны, но могут пригодиться

- _**TODO.md**_ — что делаем дальше, задачи на сегодня

**Папка docs/** — здесь всё про то, как проект устроен:

- _**workflows/**__ _— описания автоматизаций (что делают, как работают)

- _**architecture/**_ — архитектура, описание как все работает и связано

- _**resources/**_ — дополнительные материалы, полезные ссылки итд

![image](https://ajvmyjuqfdjrljmodbdb.supabase.co/storage/v1/object/public/lesson-media/uploads/1767308779732-Screenshot_2026-01-02_at_02.06.16.png)

**Папка prompts/** — промпты для агентов. Каждый файл это один агент:

- _Копирайтер_

- _Генератор картинок_

- _Анализатор данных_

Когда промпты лежат в одном конкретном месте, легче их находить и редактировать (а редактировать промпты тебе придется раз двести, пока не доведёшь до идеала)

💡 **Это не единственный способ:
**Структура зависит от проекта. Маленький проект = хватит свободного формата. Большой = потребуется продуманная организация файлов и документов. Главное, чтобы ты сам понимал где что лежит, и чтобы твоему агенту (!) было удобно ориентироваться в этой структуре (его можно прям спросить: «тебе комфортно, бро?»).

## **[ CLAUDE.md — КОНТЕКСТ ДЛЯ АГЕНТА ]**

В папке проекта можно создать файл CLAUDE.md — агент читает его автоматически и следует написанным там правилам.

**Минимальный пример:**

```markdown
# Проект: Telegram-бот для магазина

База: Airtable
Автоматизация: принимает заказы, записывает в базу, уведомляет менеджера
```

Теперь Claude сразу понимает контекст. Не нужно объяснять каждый раз.

💡 **Это важная тема:
**Правильно оформленный CLAUDE.md делает агента эффективнее в разы. Файл должен быть коротким и структурированным — есть конкретные правила как его писать. Подробно про оформление и примеры — в курсе "Claude Code", урок "Файл CLAUDE.md".

⚠️ **Тоже важно:
**Без CLAUDE.md агент каждый раз спрашивает или пытается понять _"а что это за проект?"_. А с ним сразу понимает как в проекте всё устроено..

## **[ ЭКСПОРТ ИЗ N8N ]**

Чтобы Claude увидел твою автоматизацию:

1. Открой workflow в N8N

1. Три точки справа сверху → Download

1. Сохрани в папку проекта

Теперь можешь спросить:

```text
Посмотри мой workflow. Что можно улучшить?
```

Claude откроет JSON, разберётся в структуре, ответит.

💡 **Это ручной способ:
**Скачивать JSON каждый раз — рабочий вариант для старта. Но в следующем модуле мы подключим N8N-MCP, и агент получит прямой доступ к твоему N8N: сможет видеть все workflow, шаблоны и ошибки без ручного экспорта.

## **[ ПРОВЕРКА ]**

```text
Какие файлы ты видишь в папке?
```

Если перечислил — всё работает.
