## **[ ЗАЧЕМ ЭТО НУЖНО ]**

По умолчанию Claude спрашивает разрешение на каждое действие:

- "Можно использовать search_nodes?"

- "Можно проверить workflow?"

- "Можно открыть файл?"

_Это безопасно, но замедляет работу. Настроим так, чтобы безопасные вещи работали сразу, а опасные — только с подтверждением._

## **[ ГДЕ ХРАНЯТСЯ НАСТРОЙКИ ]**

Настройки хранятся в файле `settings.json`. Но папка скрыта.

**На Mac:**

- Путь: `~/.claude/settings.json`

- Папки, начинающиеся с точки, скрыты по умолчанию

**Как показать скрытые файлы на Mac:**

1. Открой Finder

1. Нажми **Cmd + Shift + .** (точка)

1. Скрытые папки станут видимыми (полупрозрачные)

**Или через меню:**

Finder → Go → Go to Folder → введи `~/.claude/`

**На Windows:**

- Путь: `%USERPROFILE%\.claude\settings.json`

- `%USERPROFILE%` — это папка твоего пользователя (обычно `C:\Users\ТвоёИмя\`)

- Проводник → Вид → Скрытые элементы (галочка)

💡 **Проще через агента:
**Напиши: _"Найди файл, где настраиваются права доступа для N8N-MCP. Покажи что там сейчас."
_Агент сам найдёт нужный файл и покажет содержимое.

## **[ ФОРМАТ ПРАВИЛ ]**

Файл `settings.json` — обычный JSON. Внутри секция `permissions`:

```json
{
  "permissions": {
"allow": [
  "mcp__n8n-mcp__search*",
  "mcp__n8n-mcp__get*"
],
"ask": [
  "mcp__n8n-mcp__create*"
],
"deny": [
  "mcp__n8n-mcp__delete*"
]
  }
}
```

**Как читать:**

- `mcp__n8n-mcp__` — префикс для всех команд N8N-MCP

- `search*` — звёздочка означает "всё что начинается с search"

- `search_nodes`, `search_templates` — попадут под это правило

## **[ РЕКОМЕНДУЕМЫЕ НАСТРОЙКИ ]**

Для N8N-MCP я рекомендую такую конфигурацию:

```json
{
  "permissions": {
"allow": [
  "mcp__n8n-mcp__search*",
  "mcp__n8n-mcp__get*",
  "mcp__n8n-mcp__list*",
  "mcp__n8n-mcp__validate*"
],
"ask": [
  "mcp__n8n-mcp__create*",
  "mcp__n8n-mcp__update*",
  "mcp__n8n-mcp__execute*"
],
"deny": [
  "mcp__n8n-mcp__delete*"
]
  }
}
```

**Что это значит:**

- **Поиск, чтение, валидация** — сразу, без вопросов (безопасно)

- **Создание, изменение, запуск** — спросит (пусть подтвердишь)

- **Удаление** — запрещено (защита от случайностей)

💡 **Про validate* и другие инструменты:
**Команда `validate*` проверяет workflow на ошибки — полезно разрешить без спроса. Полный список инструментов N8N-MCP: https://github.com/czlonkowski/n8n-mcp — там же актуальные примеры настроек. Твой набор permissions может отличаться в зависимости от проекта.

⚠️ **После изменения настроек:
**Сделай Reload Window (Cmd/Ctrl+Shift+P → Developer: Reload Window). Без этого изменения не применятся.

## **[ НЕ ПОНИМАЕШЬ — СПРОСИ АГЕНТА ]**

Вся эта техническая информация может выглядеть сложно. JSON, пути к файлам, звёздочки... Но не парься. Забей.

Если видишь сложную инструкцию — просто скинь её агенту:

```text
Вот инструкция по настройке прав для N8N-MCP:
[вставляешь текст или ссылку]

Объясни что тут написано и помоги настроить.
```

Агент прочитает, разберётся и объяснит человеческим языком. А потом поможет применить — шаг за шагом.

💡 **Это работает для любых технических инструкций:
**Нашёл документацию на английском? Скинь агенту. Непонятный error? Скинь агенту. Он разберётся и объяснит.

## **[ КАК ПРОВЕРИТЬ ТЕКУЩИЕ ПРАВА ]**

Два способа:

**Способ 1 — команда:**

```text
/permissions
```

Покажет все активные правила.

**Способ 2 — спроси агента:**

```text
Какие у тебя сейчас права? Что можешь делать без спроса?
```

## **[ ЕСЛИ ХОЧЕШЬ ПРОЩЕ ]**

Разрешить всё для N8N-MCP одной строкой:

```json
{
  "permissions": {
"allow": [
  "mcp__n8n-mcp__*"
]
  }
}
```

Звёздочка после `n8n-mcp__` — все команды без вопросов.

⚠️ **Только если доверяешь:
**Это даёт агенту полный доступ к N8N. Удобно, но если он ошибётся — исправлять тебе.

**Проверка:**

✅ Понимаешь три уровня: allow / ask / deny

✅ Знаешь где файл настроек

✅ Умеешь показать скрытые папки на Mac

✅ Настроил права для N8N-MCP
