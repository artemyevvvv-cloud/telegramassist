# Голосовые и медиа

**Курс:** constructor  
**URL:** https://www.aizdec.me/courses/constructor/voice-media

---

**Система принимает **_голосовые сообщения, фото, видео, документы, стикеры и GIF_. Каждый тип обрабатывается по-своему: голосовые расшифровываются через Deepgram или Whisper, фото анализируются через Claude Vision, видео разбирается на аудио и ключевые кадры.

---

## Распознавание речи

Два провайдера на выбор:

**[ x ] Deepgram
**Облачный сервис, модель Nova-3, русский язык. Задержка 1-2 секунды. $200 бесплатных кредитов при регистрации (console.deepgram.com), далее ~$0.004/мин. Подключение: ⚙️ → 🔑 → `DEEPGRAM_API_KEY`.

**[ x ]  Whisper
**Работает локально на сервере, модель ggml-small-q5_1 (квантизированная). Бесплатно, без внешних API. Медленнее Deepgram, но не зависит от сети. Если ни один провайдер не настроен, при отправке голосового бот предложит установить Whisper кнопкой (~8 минут, делается один раз).

Когда подключены оба, Deepgram используется по умолчанию (выше точность). Переключение: ⚙️ → Дополнительно → 🎤 Голосовые.

🎙️ **Обработка голосового:** бот скачивает OGG-файл из Telegram, конвертирует в WAV 16kHz (ffmpeg), отправляет провайдеру. Результат показывается в цитате (первые 300 символов) перед обработкой как обычного текста. Для записей длиннее 30 секунд показывается примерное время обработки.

# Фото

Отправляешь фото = агент анализирует через Claude Vision. Распознаёт текст на изображении, понимает контекст (скриншот ошибки, фото доски, инфографика). Входит в стоимость запроса, отдельной платы нет.

# Видео

Видео и кружочки обрабатываются параллельно по двум каналам: из аудиодорожки извлекается текст (Deepgram или Whisper), из видео = ключевые кадры (2 для кружочков, 3-5 для обычного видео). Агент получает и расшифровку, и изображения.

Лимит: 20 МБ (ограничение Telegram Bot API). Видео в формате VP9 автоматически конвертируется в H.264 перед отправкой в Telegram.

# Документы

Текстовые файлы: `.txt`, `.md`, `.json`, `.csv`, `.ts`, `.js`, `.py`, `.yaml`, `.sh`, `.log`, `.sql`, `.xml`, `.html`, `.css`. Файл сохраняется на VPS, агент читает содержимое. Лимит: 20 МБ.

# Стикеры и GIF

Статичные стикеры обрабатываются как фото через Claude Vision. Анимированные и видео-стикеры не поддерживаются.

Несколько медиа-файлов одной галереей (альбом) собираются в один запрос через буфер 2.5 секунды. GIF обрабатываются аналогично.
