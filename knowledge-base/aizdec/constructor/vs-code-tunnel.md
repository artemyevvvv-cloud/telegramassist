**Claude Code в VS Code > и агента в телеграм = можно связать.** В нашей системе это один и тот же агент, с теми же сессиями и единой памятью.

**Ты общаешься с агентом через Telegram. **Он отвечает, делает задачи, запоминает.

**А VS Code даёт второй интерфейс к тому же агенту:** ты с ноута видишь файлы, редактируешь их, работаешь с кодом или другими сложными задачами. 

---

# Подключение через тг-бота

**[1]** Напиши боту команду `/connect`

**[2]** Бот пришлёт ссылку на **github.com/login/device** и одноразовый код. Открой ссылку, введи код, подтверди.

**[3]** **Бот пришлёт прямую ссылку на VS Code в браузере. Кликни > откроется редактор, подключённый к твоему серверу.**

**Теперь разберемся с папками.**

# Установка Claude Code

Теперь ставишь расширение **Claude Code** от Anthropic. Это важно.

![image](https://ajvmyjuqfdjrljmodbdb.supabase.co/storage/v1/object/public/lesson-media/uploads/1773642027247-Screenshot_2026-03-15_at_03.35.51.png)

**[1]** В VS Code нажми иконку расширений (квадратики слева) или `Cmd+Shift+X`.

**[2]** В поиске напиши **"Claude Code" → Install.**

**[3]** Расширение попросит авторизоваться. Используй тот же аккаунт Claude, что подключён к серверу (с оплаченной подпиской).

**После этого Claude Code работает в VS Code точно так же, как в Telegram: отвечает, выполняет задачи, использует инструменты, запоминает.**

# Сессии (темы диалогов) связаны

**Поработал в VS Code над проектом → сессия сохранилась. **Открыл Telegram, написал `/sessions` → видишь эту сессию в списке. Можешь продолжить прямо оттуда.

**И наоборот: **начал задачу через Telegram, потом открыл VS Code и продолжил с того же места.

`/newtask` в Telegram — начинает новую сессию. Предыдущая не удаляется, к ней можно вернуться.

# Что можно делать в VS Code

**[ + ] Редактировать файлы
**Кликнул → открылось → правишь → `Cmd+S`. Поправил SOUL.md в VS Code — агент подхватит при следующем сообщении.

**[ + ] Работать через Claude Code
**Открыл панель Claude Code → пишешь задачу → агент работает с файлами, коммитит код, ищет в интернете. Всё то же что в Telegram, только с визуальным доступом к файлам.

**[ + ] Открывать терминал в проекте
**Внизу `` Ctrl+` `` — полноценный shell на сервере. Команды, логи, git.

**[ + ] Агент тоже пишет в эти файлы
**Если открыт MEMORY.md и агент в это время дописал туда факт, VS Code покажет что файл изменился.

**[ + ] Бля, да всё можно. Я ж об этом три бесплатных курса публиковал :)**

# Авторизация живёт ~24 часа

GitHub токен для туннеля действует примерно сутки. Это ограничение Microsoft, но к нему быстро и легко привыкаешь. Когда перестаёт работать — пишешь `/connect` заново, проходишь авторизацию (30 секунд). 

---

# Частые проблемы :/

**Ссылка от бота не открывается**

- Проверь что залогинен в GitHub в браузере тем же аккаунтом, которым проходил авторизацию

**Пусто после подключения**

- Нажми "Open Folder" → выбери `/home/client/workspace/`

**Туннель отключился**

- Токен истёк. Пиши `/connect`, пройди авторизацию заново

**Claude Code не видит файлы**

- Убедись что открыта правильная папка. Claude Code работает в контексте открытой папки
