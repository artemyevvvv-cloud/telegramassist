# Проекты: организация работы

Для задач на один запрос ~/workspace/ достаточно. Но когда работа длится дни или недели = нужна изоляция. Для этого ~/projects/.

## **Создать проект:**

→ /projects → "➕ Новый проект" → название

→ Или: "Склонируй такой-то репозиторий"

→ Или: "Создай проект для ресёрча по конкурентам"

Агент создаст папку в ~/projects/, инициализирует git, подготовит memory/ и knowledge/ для проектных заметок. Проект сразу станет активным.

### **Переключиться:**

→ /projects → выбрать нужный из списка

**При переключении рабочий контекст меняется, сессия сбрасывается. Такова задумка: другой проект = другой контекст. **Но идентичность (SOUL.md, MEMORY.md, кто ты) остаётся — она всегда из ~/workspace/.

### **Вернуться в workspace:**

→ /projects → "workspace"

---

# Что внутри проекта

```text
~/projects/мой-проект/
  memory/               ← заметки по проекту
  knowledge/            ← решения, конфиги
  src/                  ← код (или любые файлы)
  .git/                 ← свой репозиторий
```

**У каждого проекта свои memory/ и knowledge/** — заметки одного проекта не мешают другому. Git свой, коммиты и ветки изолированы.

**Глобальные знания (MEMORY.md, дневник) доступны из любого проекта.**

# Как давать задачи

Агенту нужен контекст. Чем точнее ты объяснишь что хочешь, в каких рамках действовать и как ты это видишь, тем лучше результат.

Расскажи что у тебя есть, что тебе нравится, что нет. Если есть пример — покажи. Если не можешь сформулировать точно — скажи как есть, агент поможет уточнить. Главное — дать ему рамки, в которых работать.

**Размыто:**

→ "Сделай мне сайт"

**Конкретно:**

→ "Мне нужен лендинг для моего продукта. Одна страница, три секции: что это, для кого, цена. Внизу форма заявки. Мне нравится минималистичный дизайн, тёмная тема. Вот пример сайта, который мне нравится: ..."

**Тоже нормально:**

→ "Мне нужно разобраться с конкурентами. Я толком не знаю кто на рынке, но хочу понять что предлагают и за какие деньги. Помоги структурировать"

Если ты сформулировал задачу чётко = агент предложит план и можно переходить к работе.** Если задача большая, не нужно описывать всё сразу. Дай общую картину, потом двигайтесь по шагам.**

# Три сценария

### 📱 Код

→ "Создай проект для телеграм-бота. Хочу чтобы он мониторил цены и присылал уведомления. Расскажу подробнее когда создашь"

Агент создаст проект, спросит детали, предложит структуру → ты подтвердишь → он реализует пошагово с коммитами → предложит задеплоить.

### 🔍 Ресёрч

→ "Создай проект по анализу конкурентов. Мне нужно разобраться кто есть на рынке онлайн-образования, что предлагают, какие цены"

Агент создаст проект → поищет в интернете → сохранит данные в knowledge/ по каждому конкуренту → соберёт сводную таблицу. Через неделю: "Что мы знаем про конкурента Y?" — найдёт в knowledge/.

### 📝 Контент

→ "Создай проект для блога. Мне нужно написать серию статей про AI в маркетинге. Начни с ресёрча и плана"

Агент создаст проект → исследует тему → предложит план серии → для каждой статьи: ресёрч → структура → черновик в отдельном файле.

**Проекты = не только про код. Это способ организовать любую работу, которая длится больше одного разговора.**

# Как работает над длинными задачами

Когда задача на 5+ шагов, агент работает системно:

**[1] План.** Предлагает пронумерованный список шагов. Ждёт ✔ перед выполнением (в режиме «планирование»). В режиме «авто» — показывает план и сразу работает.

**[2] Инкрементальное выполнение.** Не пишет 500 строк за раз. Делает шаг, фиксирует результат (git commit), переходит к следующему. Каждый шаг — рабочее состояние.

**[3] Прогресс.** Для задач дольше 3 минут создаёт `_progress.md`: что сделано, что дальше. Обновляет после каждого шага.

**[4] Завершение.** Удаляет `_progress.md`, подводит итог.

**Зачем это надо: **если сессия прервётся (таймаут, лимит, рестарт) = ничего не потеряно. Коммиты сохранены, `_progress.md` описывает где остановился. Напишешь "Продолжи" — агент подхватит с того же места.

# Когда что-то идёт не так

Агент не всегда делает то, что ты ожидаешь. Вот что помогает:

**/stop** — остановить прямо сейчас. Агент прервёт текущую задачу.

**/newtask** — начать новую задачу. Сессия сбрасывается, но файлы и коммиты сохраняются.

**/undo** — отменить последнюю правку файла.

**/clear** — полный сброс контекста и истории. Когда агент совсем запутался и проще начать с чистого листа.

**"Объясни что ты делаешь"** — в любой момент можно спросить. Агент опишет что делал, зачем, и какой следующий шаг.

**Режим «контроль»** (из /settings) — если хочешь видеть каждое изменение до того как оно применится. ✔ / ✖ на каждую правку файла.

**Главное:** использовать /stop и /undo — не страшно. Остановить = не значит потерять. Все коммиты на месте. Лучше остановить и скорректировать, чем дать агенту уйти не туда.

# Полезные привычки

**Давай контекст.** Че�� точнее описал задачу, тем меньше итераций.

**Используй проекты для всего длинного.** Код, ресёрч, контент, планирование. Если задача на больше чем один день — создай проект.

**Проверяй план перед выполнением.** Режим «планирование» для больших задач. Дешевле исправить план, чем переделывать.

**"Продолжи" после перерыва.** Не нужно объяснять заново. Агент восстановит контекст из `_progress.md` и git-истории.
