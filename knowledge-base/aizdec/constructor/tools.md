# Два режима

Система работает в двух режимах: подписка (Claude Pro/Max) и API (ключ Anthropic). Возможности одинаковые, механика чуть разная.

|  | Подписка | API |
| --- | --- | --- |
| **Стоимость** | Фиксированная ($20-100/мес) | За каждый запрос |
| **Лимиты** | Есть (при превышении → можно переключиться на API) | Нет (пока есть баланс) |
| **Контекст** | До 1M токенов (Claude Max) | Стандартный |

Большинство начинают с подписки. API подключается через /settings когда нужен «headless»-режим или запас на случай лимитов. Переключение между режимами: вручную через /settings или автоматически (если подписка упёрлась в лимит и API-ключ подключён, агент сам переключится).

**Дальше о том, что агент умеет. Всё это работает в обоих режимах.**

## 🌐 Интернет

**Загрузка страниц: статьи, документация, посты — агент получает текст и анализирует.**

HTTP-запросы к внешним API: если у сервиса есть API (Notion, Google Sheets, Airtable, CRM, трекер задач), агент может с ним работать. Добавь ключ через /settings, дальше агент сам разберётся.

→ "Прочитай эту статью и сделай конспект"

→ "Создай задачу в Notion через API"

`web_fetch` — загрузка и чтение страницы. `http_request` — полноценный HTTP-запрос (GET, POST, PUT, DELETE) к любому API.

## 🔍 Поиск

**В интернете.** Агент ищет, читает результаты и выдаёт сводку с источниками.

**По базе знаний.** Семантический поиск = по смыслу, не по ключевым словам. Работает через эмбеддинги OpenRouter — ключ не нужен, включено по умолчанию. Три режима: по словам, по смыслу, гибридный (оба вместе).

→ "Что я знаю про маркетинг?" — найдёт по смыслу, даже если слово «маркетинг» нигде не написано

`web_search` — поиск в интернете. `vector_search` — семантический поиск по базе знаний. `monitor_search` — поиск по данным мониторинга. `search_memory` — текстовый поиск по дневным заметкам.

## 🧠 Память

**МНОГОУРОВНЕВАЯ СИСТЕМА**

**Дневник** — быстрые заметки: что произошло, промежуточный результат. Агент записывает автоматически после каждой сессии.

**MEMORY.md** — долгосрочная память: факты, которые агент помнит всегда. Дополняется из разговоров.

**База знаний** — статьи, ресёрчи, документация. Сохраняется для будущего поиска.

→ "Запомни что клиент Z предпочитает таблицы" — через месяц спросишь "как оформить отчёт для Z", агент вспомнит

→ "Сохрани этот ресёрч в базу знаний"

`save_note` — заметка в дневник. `update_memory` — обновить MEMORY.md. `kb_save` — сохранить в базу знаний. `kb_read` — прочитать из базы. `kb_list` — перечислить всё в базе.

## 📡 Мониторинг

Отслеживание пяти платформ: YouTube, Twitter, GitHub, Telegram-каналы, RSS. На подходе Instagram, TikTok, LinkedIn. Агент собирает обновления, оценивает по твоим интересам, хранит в локальной базе с векторным ��оиском, каждое утро присылает сводку.

→ «Следи за каналом @durov в Telegram»

→ «Добавь RSS https://blog.example.com/feed»

→ «Что нового за неделю?»

Управление через 📡 Мониторинг в главном меню. Подробнее — в уроках 16-18.

`add_source` — добавить источник. `monitor_latest` — последние обновления. `monitor_search` — поиск среди собранного. `monitor_stats` — статистика. `monitor_configure` — настройка параметров. `trigger_poll` — ручной запуск сбора.

## 📁 Файлы

**Читает, создаёт, редактирует файлы на сервере. Ищет по имени и содержимому. **Всё что лежит в workspace и projects — заметки, конфиги, скрипты, документы.

→ "Открой вчерашние заметки и найди упоминания проекта X"

→ "Создай файл brief.md с описанием задачи"

`read_file` — чтение. `write_file` — создание. `edit_file` — редактирование. `search_files` — поиск по содержимому. `list_files` — список файлов в директории.

### 💻 Терминал

**Выполняет команды в shell. Установить пакет, запустить скрипт, проверить лог, посмотреть диск, запустить сборку.**

Опасные команды (`rm -rf`, `sudo`, `kill`) блокируются автоматически. API-ключи из окружения вычищены. Подробнее в уроке по безопасности.

→ "Сколько места на диске?"

→ "Установи prettier и настрой для проекта"

`shell` — выполнение команды с «sanitized»-окружением. Блокировка опасных паттернов на уровне кода + systemd «sandbox».

## 🔀 Git

**Коммиты, ветки, диффы, история. **Агент коммитит после каждого изменения файла при работе с кодом. Опасные операции (`push --force`, `reset --hard`) требуют подтверждения.

→ "Покажи что менялось за последнюю неделю"

→ "Создай ветку agent/new-feature и начни работать там"

`git` — полный доступ к git. Деструктивные команды (force push, hard reset, branch delete) эскалируются до RED-уровня (подтверждение через кнопки).

→ "Открой сайт конкурента и покажи что там"

`browse` — получить текст страницы. `browse_screenshot` — скриншот → Claude Vision для ан��лиза.

## ⚙️ Система

**Кнопки подтверждения.** Когда агенту нужен выбор — отправляет варианты. Перед существенными действиями — ✔ Продолжай / ✖ Стоп.

**Снимок.** Git-снимок перед рискованными изменениями, чтобы можно было откатить.

**Проверка возможностей.** Какие сервисы настроены, какие ключи подключены.

→ "Что ты вообще умеешь?" — покажет что подключено и что нет

`send_buttons` — кнопки в чате. `snapshot` — git-снимок. `check_capabilities` — отчёт о настроенных сервисах. `send_media` — отправка фото/файлов в чат.

## Как инструменты работают вместе

Сила не в отдельных инструментах, а в комбинациях. Один запрос = несколько инструментов за кулисами.

**"Исследуй тему X":**

Поиск → загрузка нескольких источников → анализ → сохранение в базу знаний. Руками — час, агент — 3 минуты.

**"Что нового за неделю?":**

Мониторинг → фильтрация → анализ → сводка. Агент знает кого мониторить, сам вытягивает, сам отсеивает шум.

**"Открой сайт и покажи что там":**

«Browsing» → скриншот → анализ. Полноценный просмотр.

**Попробуй:** дай агенту задачу на ресёрч "найди 3 лучших инструмента для [твоя тема]". Посмотри в футере сколько инструментов задействовал.

## Навыки и MCP

Помимо встроенных инструментов, агент использует навыки и MCP-серверы. Навыки — markdown-файлы с инструкциями. Четыре предустановлены: транскрипция видео, скачивание медиа, парсинг сайтов, поиск по базе знаний. Навыки подгружаются в контекст автоматически по ключевым словам — если тема совпадает.

MCP-серверы — подключение внешних сервисов (Supadata, Apify). Управление через 🧩 Навыки и 🔌 MCP в главном меню. Подробнее — в уроке 13.

## Что агент в этой версии НЕ может

- Не может звонить — только текст, голосовые, файлы

- Не может платить — операции с деньгами требуют подтверждения

- Не может скачать длинное видео в чате — до 20 МБ (для YouTube — отдельная механика через мониторинг)
