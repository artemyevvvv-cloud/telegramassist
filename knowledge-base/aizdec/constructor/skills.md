**Навыки** = это markdown-файлы с инструкциями, которые расширяют возможности агента. Хранятся в `~/workspace/.claude/skills/`, загружаются в контекст автоматически по ключевым словам. Четыре навыка предустановлены, остальные создаёшь сам или через агента.

# Структура на сервере 🧩

Каждый навык = папка с файлом `SKILL.md` внутри.


Существует уровня хранения:

- `~/workspace/.claude/skills/` — workspace-навыки, доступны агенту в любом проекте

- `~/workspace/<проект>/.claude/skills/` — project-навыки, доступны только когда проект активен

Отдельная папка `_always/` содержит системные навыки. Они загружаются в контекст при каждом запросе целиком. Защищены от перезаписи: ни агент, ни shell-команда не могут их изменить.

## Загрузка в контекст

Система загружает навыки избирательно:

**[1]** Из `_always/` загружается всё, каждый раз.

**[2]** Для остальных система сравнивает ключевые слова твоего сообщения с описанием навыка. Слова разбиваются на токены, служебные и короткие отфильтровываются. Если совпали 2 или больше токенов, навык загружается целиком.

**[3]** Если совпадений нет, в контекст попадает только описание (первые 250 символов). Агент знает что навык существует, но содержимое не видит.

Файлы кэшируются на 60 секунд. После изменения навыка агент подхватит обновление со следующего сообщения.

# Меню навыков

Главное меню → 🧩 Навыки.

**Папки
**Workspace и project (если активен). Рядом с каждой количество навыков.

**Список
**🧠 = активный навык, ⏸ = выключенный. 

**Карточка
**Описание, расположение, размер файла. Кнопка «📄 Содержимое» показывает полный текст SKILL.md.

## Создание навыков

**Через агента:**

→ «Создай навык для код-ревью: стиль, баги, производительность»

→ «Добавь навык для анализа конкурентов: данные с сайта, сравнение»

Агент создаёт SKILL.md в правильной папке с правильным форматом. Навык активируется сразу, без перезапуска.

**Выключить навык без удаления =** кнопка ⏸ в меню или `disabled: true` в frontmatter файла.

# Agent hooks

**Навыки = **инструкции в промпте (агент читает и следует). **Hooks = **исполняемый код (агент вызывает как инструмент).

Файлы `.mjs` в `~/.iia/agent-hooks/`. Каждый файл экспортирует функцию `install(ctx)`, которая через `ctx.pushTool()` регистрирует новый инструмент. После регистрации агент может вызывать его наравне со встроенными.

```javascript
// ~/.iia/agent-hooks/10-my-crm.mjs
export default async function install(ctx) {
  ctx.pushTool({
name: "crm_check",
description: "Проверить статус клиента в CRM",
input_schema: {
  type: "object",
  properties: { client_id: { type: "string" } },
  required: ["client_id"]
},
execute: async (input) => {
  const resp = await fetch(`https://api.mycrm.com/clients/${input.client_id}`);
  return await resp.text();
}
  });
}
```

Числовой префикс в имени (`10-`, `20-`) задаёт порядок загрузки. Файлы с `_` в начале игнорируются.

При обновлениях системы папка hooks не затрагивается. Если hooks вызвали 5 последовательных крашей бота, система создаёт файл `_disabled` и отключает все hooks до ручного удаления этого файла.

💭 **От Тома:
