# Дайджест и скоринг

**Курс:** constructor  
**URL:** https://www.aizdec.me/courses/constructor/digest-scoring

---

Дайджест = утренняя сводка по собранным данным. Отдельный блок на каждый тип источника, с иконками важности и переводом на русский. Скоринг определяет какие материалы попадают в сводку, а какие отфильтровываются как шум.

---

# Утренний дайджест

Каждый день в заданный час (по умолчанию 9:00 по таймзоне из /settings) агент присылает сводку. Для каждого материала: перевод (2-3 предложения) и пояснение, почему это может быть полезно. Иконки: ♦️ critical, 🔸 important, 🔹 useful. Background и noise не попадают.

Если за текущий день ничего не найдено, сводка автоматически расширяется до 48 часов.

**Время отправки меняется через агента:** «присылай дайджест в 8 утра». Команда `/digest` генерирует сводку в любой момент, не дожидаясь расписания. Кнопки навигации позволяют смотреть сводки за предыдущие дни.

## Настройка дайджеста

📡 Мониторинг → 📨 Дайджест

На экране:

- Количество пунктов на каждый тип источника (кнопка с числом, настраивается)

- Вкл/выкл per source: ненужные типы отключаются

- Общий toggle: «⏸ Выключить» / «▶️ Включить»

---

# Скоринг

Модель Haiku 4.5 через OpenRouter оценивает каждый материал. На вход: первые 2000 символов контента. Контекст для персонализации: 500 символов из USER.md (профессия, задачи, интересы). Результат = категория (critical/important/useful/background/noise), причина, краткое описание. Таймаут: 30 секунд на материал.

# Критерии отбора

Тут два уровня настройки:

**Общие критерии
**Файл `~/workspace/DIGEST-CONTEXT.md`. Применяются ко всем источникам. Пример:

```markdown
Интересует:
- AI-агенты, автоматизация, инструменты разработки
- Кейсы применения LLM в бизнесе

Не интересует:
- Крипта и трейдинг
- Мемы без связи с технологиями
```

**Создание через агента:** «создай DIGEST-CONTEXT.md с моими критериями — [что важно, что нет]».

**Per-source критерии
**Дополнительные к общим, для конкретного типа источника. Например, для Twitter: «только треды длиннее 5 постов и с конкретными примерами кода».

Настройка: 📡 → 📂 Источники → платформа → 📝 Критерии. Хранятся в `config.json` → `sourceSettings.{type}.scoringContext`.

# Ранжирование в дайджесте

Три фактора определяют порядок:

**Вес оценки
**critical = 5, important = 4, useful = 3.

**Time decay
**Свежие материалы ранжируются выше. Скорость затухания зависит от типа: Twitter и Telegram за 7 дней, YouTube и RSS за 14, GitHub за 30.

**Балансировка по авторам
**Максимум 2 материала от одного автора, чтобы один источник не забивал всю сводку.
