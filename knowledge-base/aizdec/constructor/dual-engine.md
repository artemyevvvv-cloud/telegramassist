# Два движка

**Курс:** constructor  
**URL:** https://www.aizdec.me/courses/constructor/dual-engine

---

**Система поддерживает два режима работы с Claude:** подписка (CC CLI subprocess) и прямой вызов Anthropic API. При старте бота `engine.ts` проверяет переменные окружения: `CLAUDE_CODE_OAUTH_TOKEN` → подписка, `ANTHROPIC_API_KEY` → API. Если оба на месте, подписка используется по умолчанию, API подключается как fallback.

Разберёмся как это работает детальнее.

---

# Подписка Claude

Агент запускает Claude Code CLI как subprocess: `claude -p "запрос"`. Сессия хранится на стороне Anthropic. Можно продолжить после рестарта через `--resume`.

# Автоматический fallback

Если подписка упирается в rate limit (HTTP 429), engine.ts проверяет наличие ANTHROPIC_API_KEY. Если ключ есть, запрос выполняется через API. В футере ответа пометка о fallback и стоимость запроса. Переключение прозрачное.

Если API-ключа нет, агент сообщает: «Лимит подписки исчерпан. Обновится через ~5 часов.»

**Ручное переключение: **⚙️ → 🔌 Подключение. При смене режима сессия сбрасывается (контекст между подпиской и API несовместим).
