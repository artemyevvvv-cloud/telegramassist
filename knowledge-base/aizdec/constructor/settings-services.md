## 🔌 Два режима работы

**Подписка Claude (Pro/Max).** Фиксированная цена, сессии сохраняются, можно продолжить задачу ("Продолжи"). Для подключения нужен токен: на ноутбуке выполни `claude setup-token`, скопируй результат, вставь в бота.

**API-ключ Anthropic.** Платишь за каждый запрос. Получить: console.anthropic.com → API Keys. Вставь ключ (начинается с `sk-ant-api...`) в бота.

Если подключил оба = можно переключаться одной кнопкой. При переключении сессия сбрасывается (контекст несовместим между режимами).

Не нужно подключать всё сразу. Начни с того что используешь: голосовые → Deepgram. Код → GitHub. Остальное по мере необходимости.

### **Как подключить:**

**[1]** /settings → 🔑 Переменные окружения

**[2]** Выбери сервис из списка или нажми "Вписать свою" для нестандартного ключа

**[3]** Для каждого сервиса бот покажет: где получить ключ, какой формат, какие права выбрать

**[4]** Вставь ключ. Для своих переменных можно добавить описание — агент увидит его и будет понимать зачем этот ключ. Например: добавил `MY_CRM_TOKEN` с комментарием "CRM для лидов" — в следующем запросе агент уже знает что этот ключ для работы с CRM

**[5]** Ключ сохраняется, агент начинает его использовать сразу. Перезапуск не нужен.

### **Важно про токены:**

- Для GitHub — используй «fine-grained» токен на конкретный репо, не «classic» на все

- Для Railway — токен проекта, не токен аккаунта

- Для Vercel — укажи VERCEL_PROJECT_ID чтобы ограничить агента одним проектом

- Для любого сервиса — минимальные права, дневные лимиты

**Системные ключи** (BOT_TOKEN, ANTHROPIC_API_KEY и ещё 9) защищены от перезаписи. Ни ты, ни агент не можете их случайно сломать.

**Подробные инструкции по каждому сервису есть в файле SERVICES.md на твоём сервере. Открой через VS Code или попроси агента: "покажи SERVICES.md".**

# 🧠 Модель

Какую модель использует агент для ответов.

**На подписке Claude Max:**

- **Opus 4.6** — мощнее, глубже думает. Для сложных задач, кода, анализа

- **Sonnet 4.6** — быстрее. Для простых вопросов, рутины

- **Opus 4.6 (1M)** — миллион токенов контекста. Для огромных кодовых баз и длинных сессий. Включён в подписку Max, дополнительной оплаты до лимита нет

- **Sonnet 4.6 (1M)** — миллион токенов на Sonnet. Требует включённый «Extra Usage» на claude.ai

**На API-ключе:**

- Opus 4.6 и Sonnet 4.6 (200K контекст)

**Рекомендация: **начни с Opus. Если замечаешь что агент думает слишком долго на простых задачах, переключи на Sonnet для рутины.

**При смене модели сессия сбрасывается, если что.**

# ⏳ Таймеры

Агент выполняет задачи по расписанию. Говоришь «каждое утро в 9 — сводка новостей» — он создаёт таймер и делает сам. Три типа: ежедневные, еженедельные, одноразовые. До 20 штук. Одноразовые выключаются после выполнения.

Управление: настройки → Таймеры. Список, включение/выключение, удаление. Создание — через агента естественным языком. Подробнее — в уроке 19.

**🕐 Таймзона.
**Все расписания привязаны к ней. Выбираешь из списка (Москва, Лондон, Берлин, Нью-Йорк и т.д.), бот показывает текущее время для проверки.

# 💰 Лимит расходов

Дневной лимит на API-запросы. Варианты: без лимита, $10, $25, $50, $100, $200.

**Как работает:**

- При 80% — агент предупредит один раз

- При 100% — агент приостановит работу до следующего дня

Рекомендация: $25-50 на старте. Хватает для активной работы, но не разоришься если агент зациклится на сложной задаче. **В этом разделе учитываются только вызовы через API-ключ. Подписка Claude — фиксированная, лимит на неё не влияет.**

## 🔒 Режим работы с файлами

**Три режима** — как агент обрабатывает изменения:

**😈 Авто.** Агент пишет и правит файлы свободно. Для тех кто доверяет агенту и хочет скорость. По умолчанию.

**☝🏽 Контроль.** Перед каждой правкой файла — пауза с кнопками ✔ / ✖. Максимальный контроль, но медленнее. Для критически важного кода.

**🤓 Планирование.** Агент сначала пишет пронумерованный план, ждёт ✔. После подтверждения выполняет без остановок. Для больших задач и рефакторинга.

Переключай под задачу: рутина → авто, важный код → контроль, большой проект → планирование.

## ⚙️ Дополнительно

**✏️ Детализация.** Два уровня: стандартный («читаю файл 👁️») или подробный (плюс имя файла, команда). Подробный полезен для дебага.

**🎤 Голосовые.** Выбор провайдера: Whisper (локально, бесплатно) или Deepgram (облако, точнее). Подробнее — в уроке 15.

**💾 Хранилище мониторинга.** SQLite (локально на сервере) или Supabase (облако). По умолчанию — SQLite.

**🔐 SSH доступ.** Добавить SSH-ключ, открыть порт для прямого доступа к серверу. Подробнее — в уроке 14.

**♻️ Перезапуск.** Перезагрузить бота. Занимает 5-10 секунд.

## Мониторинг источников

Мониторинг настраивается через 📡 Мониторинг в главном меню или словами агенту. Пять платформ: YouTube, Twitter, GitHub, Telegram-каналы, RSS. На подходе Instagram, TikTok, LinkedIn.

Говоришь «следи за каналом @durov» или «добавь RSS» — агент добавляет и начинает сбор. Утром приходит персональная сводка. Все данные хранятся в локальной базе с поиском по смыслу.

Для YouTube-мониторинга нужен Deepgram (расшифровка аудио). Для Twitter нужен ключ X API (через /settings → 🔑).
