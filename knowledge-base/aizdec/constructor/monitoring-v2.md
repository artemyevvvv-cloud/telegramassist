# Мониторинг

**Курс:** constructor  
**URL:** https://www.aizdec.me/courses/constructor/monitoring-v2

---

**Система мониторинга поддерживает пять типов источников: **YouTube, Twitter, GitHub, Telegram-каналы, RSS. Каждый тип собирает данные по своему расписанию. Собранные материалы проходят через скоринг (Haiku 4.5 с учётом профиля из USER.md), сохраняются в локальную SQLite-базу и попадают в утренний дайджест.

# Управление

📡 Мониторинг в главном меню.
Там четыре секции:

**📂 Источники
**Список всех каналов и аккаунтов, сгруппированных по платформе. Количество рядом с каждой. Внутри: вкл/выкл сбора, настройка критериев скоринга для этого типа.

**🔄 Сбор данных
**Интервал, время последнего сбора, количество нового. Формат: `📺 30 мин · 14:35 [+23]`. Объём хранилища и период хранения внизу.

**📨 Дайджест
**Настройки утренней сводки: количество пунктов, вкл/выкл per source, общие критерии. Подробнее в следующем уроке.

**🌐 Лента
**Просмотр собранного контента.

Сбор данных автоматический. Scheduler проверяет источники по расписанию (интервалы из таблицы выше), скорит через Haiku 4.5, сохраняет в базу.

# Скоринг

**Каждый материал оценивается по пяти уровням:** critical, important, useful, background, noise. Модель получает первые 2000 символов контента и 500 символов из USER.md для персонализации. Подробнее о критериях, ранжировании и настройке — в следующем уроке.

## Попробуй

→ Добавь один YouTube-канал и один Telegram-канал

→ Через 30 минут: `/sources` — посмотри что собралось

→ `/digest` — посмотри как выглядит сводка

💭 **От Тома:
**Чем точнее заполнен файл, отвечающий за скоринг (сортировку инфы и оценку важности), тем точнее будет собираться информация.
