# Три зоны на сервере

Всё на VPS разделено на три территории:

```yaml
~/workspace/     ← личность агента: кто он, кто ты, память, навыки
~/projects/      ← твои проекты: код, репозитории, рабочие папки
~/.iia/          ← системная зона: движок бота, ключи
```

**~/workspace/** — здесь живёт всё, что делает агента персональным. Восемь файлов ДНК (характер, профиль, память, цели), дневник, база знаний, навыки. Это центр управления. Агент загружает эти файлы каждый раз когда ты ему пишешь.

**~/projects/** — здесь живут твои проекты. Клонируешь репо, пишешь код, ведёшь документацию. Каждый проект в отдельной папке со своими memory/ и knowledge/. Контекст изолирован: заметки одного проекта не мешают другому. При этом идентичность агента (SOUL.md, USER.md, MEMORY.md) всегда берётся из workspace. Проект переключился, характер остался.

**~/.iia/** — системная зона. Движок бота, API-ключи, мониторинг. Обновления приходят от нас централизованно. Агент тоже не может сюда добраться: чтение `.env` заблокировано, код бота «read-only». Ключи добавляешь через /settings в Telegram, система сама запишет их куда нужно.

Разделение принципиальное: мы обновляем движок, твоя личность и проекты не затрагиваются. Ты меняешь (условно) SOUL.md, а движок продолжает работать.

# Что внутри workspace:

```yaml
~/workspace/
  SOUL.md              ← характер агента (тон, стиль, манера)
  USER.md              ← кто ты (профессия, контекст, стек)
  MEMORY.md            ← долгосрочная память
  GOALS.md             ← текущие цели
  MISSION.md           ← зачем ты здесь
  PROJECTS.md          ← список активных проектов
  PREFERENCES.md       ← как предпочитаешь работать
  LEARNED.md           ← выводы агента
  CLAUDE.md            ← правила поведения
  SERVICES.md          ← подключённые сервисы
  memory/              ← дневник (заметки по дням)
  knowledge/           ← база знаний (статьи, ресёрчи)
  .claude/skills/      ← навыки (markdown-файлы)
```

**Все файлы = это обычный текст. **Открываешь, читаешь, редактируешь через Telegram или через VS Code. Восемь файлов ДНК (от SOUL.md до LEARNED.md) мы подробно разбираем в уроке 2. Сейчас достаточно понять, кто что делает.

### **Ты редактируешь когда хочешь:**

SOUL.md (поменять тон), GOALS.md (обновить цели), PROJECTS.md (добавить проект), PREFERENCES.md (изменить стиль работы), MISSION.md (скорректиро��ать фокус). Через Telegram («обнови мои цели на этой неделе») или в VS Code руками.

### **Агент заполняет сам:**

LEARNED.md (выводы из сессий), memory/ (дневные заметки), knowledge/ (сохранённые статьи и ресёрчи), дополняет MEMORY.md фактами из разговоров. Ты это не контролируешь специально, просто общаешься, агент запоминает.

### **Генерируется при первом запуске:**

USER.md и SOUL.md создаются во время «bootstrap»-интервью (урок 2). Первая версия генерируется из твоих ответов. Дальше можешь менять как угодно.

### **Обновляется нами:**

CLAUDE.md и SERVICES.md содержат шаблонные секции (правила, инструкции сервисов). Мы обновляем их при выходе новых версий. Твои личные правки в этих файлах сохраняются.

# Что внутри projects:

```yaml
~/projects/
  название-проекта/
    memory/            ← заметки по проекту
    knowledge/         ← решения, конфиги
    ...                ← код, файлы, всё что нужно
```

Когда говоришь агенту_ «работаем над таким-то проектом»_, он переключает контекст на эту папку. Видит файлы проекта, помнит заметки проекта, не путает с другими. Но характер, цели и долгосрочная память по-прежнему из ~/workspace/.

**Проекты создаются по ходу работы. **Попросил агента склонировать репо или начать новый проект, он создаст папку в ~/projects/ и будет работать там. **Вручную тоже можно, через меню бота.**

## Что происходит когда ты пишешь в Telegram:

**Вот ты отправил сообщение****.** **Дальше:**

**[1] Бот получает его.** На сервере постоянно работает процесс (systemd сервис). Он подключён к Telegram и ждёт входящих.

**[2] Медиа обрабатывается.** Если это голосовое → то расшифровка через Deepgram (у него охуенная точность, бесплатный тариф на $200, и в целом он стоит своих денег, и стоит недорого). Фото → подготовка для Claude Vision. Видео → параллельно: звук на расшифровку + ключевые кадры. Обычный текст проходит этот шаг мгновенно.

**[3] Загружается контекст.** Бот собирает всё, что агент должен знать: SOUL.md, USER.md, MEMORY.md, CLAUDE.md, активные навыки, заметки за последние дни, история текущего диалога. Упаковывается в один запрос.

**[4] Запрос уходит в Claude.** Весь контекст + твоё сообщение отправляется модели. Через подписку или через API, в зависимости от режима.

**[5] Claude думает и действует.** Модель читает контекст, понимает задачу, решает какие инструменты нужны. Может вызвать несколько подряд: поискать в интернете → скачать страницу → сохранить заметку. Каждый вызов это отдельный цикл «решил → выполнил → прочитал результат → решил что дальше».

**[6] Ответ стримится в чат.** Текст появляется в реальном времени. Длинные ответы разбиваются на несколько сообщений автоматически.

**[7] Футер.** В конце каждого ответа: время, стоимость, процент из кэша, количество использованных инструментов.

**Вот почему простой вопрос занимает 5 секунд, а **_**«исследуй тему X»**_** может занять минуту. Разница в шаге 5: сколько инструментов агент решил задействовать.**

# Два режима работы

Система может работать в двух режимах. Можно не выбирать, и использовать оба, они дополняют друг друга.

**[Подписка]
**Агент работает через подписку Claude Pro или Max. Фиксированная цена, без подсчёта токенов. Для повседневного общения оптимально. Большинство начинают с этого режима.

**[API]
**Прямые вызовы Anthropic API по ключу. Платишь за каждый запрос, но получаешь полный набор кастомных инструментов и «headless»-режим (агент работает без тебя). Подключается через /settings в один шаг.

_Если подписка упирается в лимит, система автоматически переключится на API (когда ключ подключён). Один режим упал, второй подхватил. Какие инструменты доступны в каждом режиме, разбираем в уроке 5._

# Как прилетают обновления 🛸

**Движок бота обновляется централизованно.** Мы выкатываем новую версию, она приходит на сервер. Бот перезапускается (10-15 секунд), дальше работает как обычно. Затрагивает только ~/.iia/bot/. Твои файлы в workspace и projects не меняются.

**Навыки приходят из нашего каталога.** Новые появляются в `.claude/skills/`. Можешь удалить ненужные или написать свои.

**CLAUDE.md и SERVICES.md могут получать обновлённые секции, **но твои правки сохраняются.

💭 **От Тома:
Три зоны, одно правило:** workspace = твой рабочий стол, projects = твой код, .iia = наша 🛠️ забота. Обновления не ломают настройки. Агент учится и запоминает сам, но ты в любой момент можешь открыть любой файл и поправить руками. Полный контроль 🤝🏾 без необходимости контролировать каждый шаг 😈
