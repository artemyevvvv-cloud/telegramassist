# Доступ к серверу

**Курс:** constructor  
**URL:** https://www.aizdec.me/courses/constructor/server-access

---

**Есть четыре способа подключиться к серверу: **Telegram (чат с агентом), VS Code tunnel (редактор в браузере), SSH (терминал напрямую), Recovery Console (аварийный через DigitalOcean). Каждый следующий даёт больше контроля и нужен реже.

---

# Telegram

Основной способ работы. Пишешь агенту, он выполняет задачи на сервере: файлы, код, shell-команды, мониторинг, память. Для большинства задач достаточно.

---

# VS Code tunnel

Для работы с кодом, длинных сессий, прямого редактирования файлов. Агент и VS Code работают на одном сервере с общими файлами и памятью.

**Подключение через /connect:**

**[1]** Бот запускает GitHub device flow и показывает ссылку github.com/login/device с кодом авторизации.

**[2]** Вводишь код в браузере, подтверждаешь доступ.

**[3]** Получаешь ссылку `vscode.dev/tunnel/jarvis-XXXXXXXX`. VS Code открывается в браузере с файлами сервера.

**Для десктопного VS Code: **Cmd+Shift+P → Remote-Tunnels: Connect to Tunnel → имя туннеля. Нужно предварительно активировать расширение.

# SSH

Прямой доступ к терминалу сервера. Логи, установка пакетов, редактирование системных файлов, диагностика. **Порт 22 закрыт по умолчанию и открывается только через бота.**

⚙️ Настройки → Дополнительно → 🔐 SSH:

**[1]** Создать SSH-ключ (если нет): `ssh-keygen -t ed25519`

**[2]** «🔑 Добавить SSH ключ» — вставить публичный ключ (`ssh-ed25519 ...` или `ssh-rsa ...`)

**[3]** Узнать свой IP: открыть ipv4.icanhazip.com в браузере

**[4]** «🌐 Открыть IP» — ввести IP. Фаервол откроет порт 22 для этого адреса

**[5]** Подключиться: `ssh client@IP_СЕРВЕРА`

При смене IP (переезд, перезагрузка роутера) — добавить новый через ту же кнопку. «📋 Показать ключи» и «👁 Показать IP» — для просмотра и удаления.

**Полезные команды после подключения:**

- `systemctl status iia-bot` — статус бота

- `journalctl -u iia-bot --since "1h ago"` — логи за последний час

- `df -h` — свободное место на диске

# Recovery Console

Когда бот не отвечает, VS Code не подключается и SSH не работает одновременно, спасает оно. Ввведи в боат команду: **/recovery** >>> и бот отправляет файл `recovery-access.txt` с паролем для DO Recovery Console.

**Как использовать:** cloud.digitalocean.com → Droplets → твой сервер → Access → Recovery Console. Логин и пароль из файла. После входа: `sudo systemctl restart iia-bot` для перезапуска бота.

⚠️ **Важно:** выполни /recovery сейчас и сохрани файл. Когда бот перестанет отвечать, получить файл будет невозможно.

На сервере также лежит `SERVER-ACCESS.md` с полной документацией по всем способам доступа и аварийным процедурам.
