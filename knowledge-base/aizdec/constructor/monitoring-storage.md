# Хранилище мониторинга

**Курс:** constructor  
**URL:** https://www.aizdec.me/courses/constructor/monitoring-storage

---

**Данные мониторинга хранятся в SQLite-базе на сервере:** `~/.iia/monitor/knowledge.db`. Каждый собранный материал = запись с источником, текстом, оценкой и векторным представлением. Есть три типа поиска: по словам, по смыслу и гибридный. Supabase или другое облачное хранилище для мониторинга не обязательны.

---

# Три типа поиска

**[1] Keyword
**Полнотекстовый через FTS5. Находит точные совпадения и словоформы.

→ «Найди в мониторинге упоминания Claude Code»

**[2] Semantic
**Текст превращается в вектор из 1536 чисел (модель text-embedding-3-small через OpenRouter). Поиск сравнивает векторы по близости. «Автоматизация рутины» и «AI заменяет ручной труд» — разные слова, близкий смысл.

→ «Что в мониторинге про автоматизацию бизнес-процессов?»

**[3] Hybrid
**Комбинация keyword и semantic через алгоритм Reciprocal Rank Fusion (k=60). Порог cosine similarity: 0.3. Используется по умолчанию.

Для семантического 🧠 поиска нужен `OPENROUTER_API_KEY` в переменных окружения. Эмбеддинги генерируются автоматически при сборе данных. Стоимость: ~$0.007 за 1700 материалов.

**Переключение хранилища:
**⚙️ → Дополнительно → 💾 Хранилище. SQLite (локально, по умолчанию) или Supabase (облако, если нужен доступ из других приложений).

# Транскрипция видео

YouTube-видео транскрибируются автоматически при сборе. Цепочка:

**[1]** Supadata API — мгновенно, $0.01/видео. YouTube, TikTok, Instagram, X, Facebook. Видео длиннее 20 минут через Supadata пропускаются (async job).

**[2]** Если Supadata не получил текст — Apify скачивает аудио (~$0.05), Deepgram Nova-3 расшифровывает ($0.006/мин).

Транскрипт сохраняется в поле `content`, индексируется FTS5 и получает эмбеддинг. В дайджесте отображается содержание видео, а не только заголовок.

Для работы транскрипции нужен MCP Supadata (урок 14). Без него в базу попадут только заголовок и описание видео.

# Хранение и очистка

- Retention: 30 дней (настраивается в `~/.iia/monitor/config.json`)

- Лимит диска: 500 МБ

- Очистка: автоматическая, ночью (3-5 AM). Старые записи удаляются, место освобождается

**Статистика:** 📡 → 🔄 Сбор данных — объём хранилища и период хранения внизу экрана.

## Попробуй

→ «Что в мониторинге про [твоя тема]?» — проверь семантический поиск

→ «Что нового за последние 3 дня?» — свежие материалы из всех источников

→ «Что писал @[автор]?» — поиск по конкретному человеку

💭 **От Тома:
**Все данные на твоём сервере, в одном файле SQLite. Через месяц мониторинга там будут тысячи записей с эмбеддингами, но система от этого станет только точнее. В следующих уроках мы также рассмотрим различные подходы к памяти и обработке данных в хранилище.
