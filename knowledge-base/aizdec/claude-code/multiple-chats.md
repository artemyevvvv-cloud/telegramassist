# Несколько чатов

**Курс:** Claude Code  
**URL:** https://www.aizdec.me/courses/claude-code/multiple-chats

---

Один чат это хорошо. Но группа параллельных чатов это совсем другой уровень. Можно делать несколько вещей одновременно, перепроверять идеи по несколько раз, и ускорять разработку.

## **[ ЗАЧЕМ НЕСКОЛЬКО ЧАТОВ ]**

**1. Разделение задач**

Один чат = одна задача. Не путаешь контексты, легче вернуться к нужному разговору.

**2. Параллельная работа**

Пока один чат работает над задачей, ты можешь в другом делать что-то ещё. Два чата могут одновременно делать разные части проекта.

**3. Перепроверка**

Сомневаешься в ответе? Открой новый чат и спроси то же самое по-другому. Сравни ответы.

**4. Разные роли**

Один чат — исполнитель, делает код. Другой — критик, проверяет что сделано.

## **[ МОЙ СПОСОБ: ТРИ ВКЛАДКИ ]**

![image](https://ajvmyjuqfdjrljmodbdb.supabase.co/storage/v1/object/public/lesson-media/uploads/1766449647615-Screenshot_2025-12-23_at_03.27.22.png)

Я обычно держу минимум три чата:

**Чат 1 — Общение**

Здесь обсуждаю идеи, задаю вопросы, планирую. Не делаю — думаю.

**Чат 2 — Работа**

Здесь Claude создаёт и редактирует файлы. Основная работа.

**Чат 3 — Проверка**

Здесь проверяю то что сделано. Прошу посмотреть критически. Ищу ошибки.

Иногда все три работают параллельно над разными частями проекта.

💡 **Напоминалка:
**Название чата создаётся автоматически из твоего первого сообщения. Поэтому начинай чат с понятной фразы: _"Работаем над формой регистрации"_ или _"Ищем баг в авторизации"_ — так будет проще найти нужный чат в истории.

---

## **[ КАК ОТКРЫТЬ НОВЫЙ ЧАТ ]**

Открой ещё одну панель Claude Code — это и будет новый чат.

**Способ 1:** Shift + Cmd/Ctrl + Escape

**Способ 2:** Кликни на иконку Claude Code в боковой панели

Старый чат никуда не денется, он в истории. Переключайся между ними.

## **[ КОГДА НАЧИНАТЬ НОВЫЙ ЧАТ ]**

**Да, новый чат:**

- Новая задача, не связанная с текущей

- Текущий чат стал слишком длинным

- Хочешь проверить решение независимо

- Чат "запутался" — проще начать заново

**Нет, продолжай в текущем:**

- Уточняешь последний ответ

- Задача связана с предыдущими

- Claude нужен контекст из истории

⚠️ **Важно:
**Длинные чаты, даже при сжатии контекста, оказывают нагрузку на комп. Много длинных чатов – это не окей. Лучше несколько, но коротких. Привыкни к этому режиму. 

## **[ ОЧИСТКА КОНТЕКСТА ]**

Если хочешь начать заново в том же чате — используй **/clear**.

Просто напиши `/clear` — чат очистится. Claude забудет всё что было раньше. Это как новый чат, но без создания нового.

Полезно когда:

- Переключаешься на совсем другую задачу

- Чат "замусорен" старыми обсуждениями

## **[ ПАРАЛЛЕЛЬНАЯ РАБОТА ]**

Можно запустить несколько Claude одновременно:

1. **В разных панелях VS Code**

Открой несколько терминалов/панелей, в каждой свой чат

1. **На разных частях проекта**

Один чат работает над страницей, другой над API — не мешают друг другу

1. **С разными задачами**

Один пишет код, другой пишет тесты, третий документацию

💭 **От Тома:
**Я никогда не туплю пока жду ответ. В одном чате Claude делает форму, во втором я проверяю предыдущую работу, в третьем планирую следующий шаг.

## **[ ПЕРЕПРОВЕРКА В НОВОМ ЧАТЕ ]**

**Полезгый приём: проверить решение независимо.**

1. Claude в чате 1 сделал что-то

1. Открываешь чат 2 (новый, чистый)

1. Показываешь результат и спрашиваешь: "Посмотри критически. Какие проблемы?"

_Чат 2 не знает контекст чата 1. Он смотрит свежим взглядом. Часто находит то, что чат 1 пропустил._

## **[ ПРАКТИКА ]**

**Попробуй прямо сейчас:**

1. Открой новую панель Claude Code (Cmd/Ctrl + Shift + P → _"Claude Code"_)

1. Напиши там что-нибудь простое

1. Посмотри в историю чатов — старый чат на месте

**Это всё. Теперь у тебя два чата, можешь переключаться между ними.**

💭 **От Тома:
**Часто, когда агент сделал работу, я сначала даже не читаю ответ, а первым делом заставляю его всё перепроверить. Или открываю соседний чат и перепроверяю там (если задача сложная). Это нормальный рабочий процесс. Доверяй, но проверяй ;)
