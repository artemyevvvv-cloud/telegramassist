# 4. Подключаем Supabase к проекту

База создана. Таблица есть. Ключи сохранены в `.env.local`.

Теперь сделаем так, чтобы твоё приложение могло общаться с Supabase — загружать и сохранять данные.

## **[ ЧТО ТАКОЕ БИБЛИОТЕКА ]**

Чтобы приложение могло "разговаривать" с Supabase, нужен переводчик. Этот переводчик — **библиотека**.

**Библиотека** — это набор готового кода, который написали разработчики Supabase. Она умеет:

- Отправлять запросы в базу данных

- Получать ответы

- Обрабатывать ошибки

Без библиотеки тебе пришлось бы писать всё это самому. С библиотекой — просто говоришь "дай все задачи" и она делает.

💡 **Аналогия:**
Библиотека — как приложение-переводчик на телефоне. Ты говоришь по-русски, оно переводит на язык который понимает база данных.

## **[ ШАГ 1: УСТАНАВЛИВАЕМ БИБЛИОТЕКУ ]**

Скажи Claude Code:

"Мне нужно подключить Supabase к проекту. Установи библиотеку @supabase/supabase-js"
**Что сделает Claude:**

- Выполнит команду установки

- Библиотека загрузится из интернета

- Добавится в твой проект

**Как проверить:**

Открой файл `package.json`. Найди раздел `"dependencies"`. Там должна появиться строка:

```text
"@supabase/supabase-js": "..."
```

Это значит — библиотека установлена. Теперь приложение умеет разговаривать с Supabase.

## **[ ШАГ 2: НАСТРАИВАЕМ ПОДКЛЮЧЕНИЕ ]**

Библиотеке нужно знать куда подключаться. Для этого создадим файл с настройками.

Скажи Claude:

"Создай файл для подключения к Supabase. Ключи лежат в .env.local: VITE_SUPABASE_URL и VITE_SUPABASE_ANON_KEY"
**Что сделает Claude:**

- Создаст файл (обычно `src/lib/supabase.ts` или похожий)

- В файле будет код который берёт ключи из `.env.local`

- И создаёт "клиент" — объект через который идёт общение с базой

**Что будет в файле (примерно):**

```typescript
import { createClient } from '@supabase/supabase-js'

const supabaseUrl = import.meta.env.VITE_SUPABASE_URL
const supabaseKey = import.meta.env.VITE_SUPABASE_ANON_KEY

export const supabase = createClient(supabaseUrl, supabaseKey)
```

Теперь в любом месте приложения можно написать `supabase.from('tasks')...` и получить данные.

## **[ ШАГ 3: ПРОВЕРЯЕМ ПОДКЛЮЧЕНИЕ ]**

Убедимся что приложение видит данные из базы.

Скажи Claude:

"Хочу проверить что Supabase работает. Сделай кнопку которая загрузит данные из таблицы tasks и покажет их в консоли браузера."
Теперь:

1. Открой приложение в браузере (обычно http://localhost:5173)

1. Открой консоль браузера:
- **Mac:** Cmd + Option + J
- **Windows:** F12, затем вкладка Console

1. Нажми кнопку

**Что должно появиться в консоли:**

✅ Данные из таблицы — например:

```text
[{id: "...", title: "Моя первая задача", ...}]
```

Или пустой массив `[]` если записей нет — тоже нормально.

## **[ ЕСЛИ ОШИБКА ]**

**"Invalid API key"**

- Проверь ключи в `.env.local`

- Убедись что названия переменных точно `VITE_SUPABASE_URL` и `VITE_SUPABASE_ANON_KEY`

- Перезапусти приложение (остановить и запустить заново)

**"relation does not exist"**

- Таблица не создана. Вернись к Уроку 3.

**Другая ошибка**

- Скопируй текст → отправь Claude

⚠️ **Важно:**
После изменения `.env.local` нужно перезапустить приложение. Просто обновить страницу не поможет — переменные окружения читаются только при запуске.
