**Напомню что мы делаем:** простое приложение для ведения задач. Пользователь добавляет задачу, и она сохраняется в базу. В прошлом уроке проверили подключение: нажали кнопку, данные появились в консоли браузера.

## **[ ПРО БЕЗОПАСНОСТЬ ]**

В Supabase есть два типа ключей (ты получил их в уроке 1):

![image](https://ajvmyjuqfdjrljmodbdb.supabase.co/storage/v1/object/public/lesson-media/uploads/1766453987395-Screenshot_2025-12-23_at_04.39.44.png)

**anon key** — для браузера

- Его видят все пользователи (он в коде сайта)

- Даёт ограниченный доступ

- Безопасно использовать в коде сайта

- Сейчас в твоём `.env.local`

**service_role key** — для сервера

- Даёт полный доступ ко всей базе

- **НИКОГДА** не используй в браузере

- Только в серверном коде (например, в webhook)

**Важно понимать:
**Сейчас любой человек может читать и писать в твою таблицу, потому что мы ещё не настроили правила доступа.

Это нормально для учебного проекта. Когда будешь делать реальное приложение, настроишь защиту в Модуле 5 (RLS). Тогда каждый пользователь будет видеть только свои задачи.
