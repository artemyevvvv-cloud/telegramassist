# Создаём проект в Supabase

**Курс:** Публикация проекта в инет  
**URL:** https://www.aizdec.me/courses/publish/create-supabase-project

---

Сейчас создадим аккаунт и проект в Supabase, внутри которого будут храниться важные данные.  Сначала получим ключи для подключения. Это займёт 5 минут. Нужен только аккаунт GitHub.

## **[ ШАГ 1: РЕГИСТРАЦИЯ ]**

1. Открой **supabase.com**

1. Нажми кнопку входа (обычно "Start your project")

1. Выбери **"Continue with GitHub"**

1. Разреши доступ

✅ Ты внутри, если видишь страницу с проектами (пока пустую).

💡 **Кнопка не там?
**Интерфейс иногда меняется. Если где-то тупишь, сделай скриншот → отправь Claude → и спроси его то, что тебя интересует.

## **[ ШАГ 2: СОЗДАЁМ ПРОЕКТ ]**

1. Нажми **"New Project"**

1. Заполни:

![image](https://ajvmyjuqfdjrljmodbdb.supabase.co/storage/v1/object/public/lesson-media/uploads/1766453355098-Screenshot_2025-12-23_at_04.27.14.png)

**Name** — любое название латиницей, например `my-app`

**Database Password** — нажми кнопку **Generate** (Supabase сам создаст сложный пароль). **Скопируй и сохрани** — понадобится позже.

**Region** — выбери ближайший к твоим пользователям. Для СНГ обычно подходит **Frankfurt**.

1. Нажми **"Create new project"**

1. Подожди минуту-две пока создаётся&

✅ Готово, когда видишь страницу проекта с меню слева.

💡 **Про бесплатный план:
**Если проект не используется неделю — Supabase его "усыпляет" чтобы экономить ресурсы. При первом запросе проект просыпается за минуту. Данные не теряются.

---

## **[ ШАГ 3: ПОЛУЧАЕМ КЛЮЧИ ]**

Чтобы твоё приложение могло подключиться к Supabase — нужны два значения:

- **URL** — адрес твоей базы данных в интернете

- **Ключ** — пропуск, чтобы база пустила твоё приложение

**Где взять:**

1. В меню слева найди шестерёнку (**Project Settings**)

1. Открой раздел **"API"**

1. Скопируй **Project URL**

1. Перейди в **Data API** и скопируй ключ **anon public**

![image](https://ajvmyjuqfdjrljmodbdb.supabase.co/storage/v1/object/public/lesson-media/uploads/1766453650861-Screenshot_2025-12-23_at_04.34.07.png)

⚠️ **Сохрани ключи в надёжное место** — менеджер паролей или защищённую заметку. Они понадобятся для настройки проекта и при деплое (публикации проекта в интернет). Заходить каждый раз на сайт, чтобы их скопировать, не оч удобно честно говоря. 

## **[ ШАГ 4: СОХРАНЯЕМ КЛЮЧИ В ПРОЕКТ ]**

Теперь добавим ключи в твой проект. Напиши Claude Code:

"Создай файл .env.local с переменными VITE_SUPABASE_URL и VITE_SUPABASE_ANON_KEY"

**Что сделает Claude:**

- Создаст файл `.env.local` в папке проекта

- Этот файл хранит секретные настройки

- Git автоматически его игнорирует — ключи не попадут в интернет

Открой созданный файл и вставь свои значения вместо подсказок.

Сохрани: **Cmd+S** (Mac) или **Ctrl+S** (Windows).
