## Что ещё за скилы 🗿

В предыдущем блоке ты создавал команды — файлы с инструкциями, которые запускаются через `/название`. Теперь пора добавить к ним настройки.

![image](https://api.aizdec.app/storage/v1/object/public/lesson-media/uploads/1772122042032-image.png)

С 2026 года команды и скиллы это одна система. Разница в том, что у скилла есть шапка: какую модель использовать, запускать ли в изолированном режиме, какие инструменты разрешить. Проще говоря: **команда без шапки = это команда. Команда с шапкой = это скилл.**

Примеры правил внутри скилла:

- Когда пишешь пост — начинай с вопроса, заканчивай призывом к действию

- Когда отвечаешь клиенту — делай это без воды, максимум 3 абзаца

- Когда проверяешь код — основной акцент на безопасности данных

_Скилы можно создавать самому, скачивать готовые, обмениваться с близкими, продавать за деньги, и не только. Их легко устанавливать и редактировать, а значит, можно точно настроить работу с Claude Code под необходимые задачи._

## Скилы + команды = збс

Вот, например, у меня есть тг-бот. Туда часто пишут люди: кто-то приходит за уроками, кто-то просит помощь, кто-то предлагает сотрудничество. С разными людьми нужен разный стиль общения.

Я начал с того, что сделал команду `/inbox` которая обрабатывает все входящие. Внутри команды прописан алгоритм:

```text
1. Загрузи новые сообщения
2. Покажи список по категориям
3. Для каждого — предложи ответ
4. Отправь после получения подтверждения
```

А ещё там прописаны правила ответов. Как общаться с теми, кто хочет сотрудничать. Что отвечать тем, кто несёт херню. Как помочь тем, у кого конкретный вопрос. Ну и так далее.

Вот это и есть скиллы 💪🏽

## Как активировать скилл 🧠

Скиллы вызываются через `/название` — точно так же как команды. Вообще, по задумке :) Клод сам должен вспоминать когда применить скилл. Иногда это работает, иногда нет (: Поэтому лучше привязывать скиллы к конкретным командам, и организовать свой рабочий процесс как набор команд. Попроси Claude настроить это, он разложит всё по файлам.

## Какие есть реальные кейсы 

**Маркетолог****: 10 скиллов для 10 типов контента **

Челик каждый день делал день посты, рассылки, рекламу. И каждый раз объяснял агенту тон, аудиторию, какие слова использовать. Тратил уйму времени на жеванину. А потом создал скиллы под каждый тип задач. И теперь по команде _"напиши пост",_ Claude Code пишет ему пиздатые посты. Среднее качество выполнения задач выросло с 20% до (!) 80%.

**Аудит клиента****: 6 часов → 20 минут**

Товарищ внедряет бизнесам автоматизации, и с каждым новым клиентом, перед разработкой проводит глубокий аудит. Изучает тон и стиль бренда, их текущие разработки, организацию используемых систем. Так вот после того, как он прописал в Claude Code команды, настроил MCP, и создал SKILLS – он буквально молится на меня, потому что 6-часовая задача стала выполняться за ~20 мин.

**Еженедельная рассылка****: 2 часа → 10 минут**

Щас у каждого уважающего себя бизнеса, есть рассылка с полезными / интересными / актуальными материалами и подборками, помогающая быть с клиентами на связи и напоминать им о новых акциях и товарах. Так вот с Клодом её ведение не просто упрощается в разы, но ещё и превращается в интересный квест. Потому что задача перестаёт быть скучной, и становится прикольной и быстрой.

# Какие бывают скилы

**[глобальные]** — работают во всех проектах.

**[проектные]** — только в конкретном проекте, правила для клиентов конкретного бота, например.

**Скажи Claude куда положить** 🗄️ и он разместит в нужном месте. Так правила для разных проектов и задач не смешаются, и будут нормально организованы.

## Связка инструментов 🧳

Итак, теперь, у нас уже есть:

**[ + ] Команды и скиллы** — инструкции для Claude.

Команды для простых задач, скиллы когда нужны продвинутые настройки. Всё вызывается через `/название`.

**[ + ] MCP** — способ соединения с внешними сервисами: ботами, таблицами, базами данных, приложениями.

**Вместе они дают мощную связку: скилл задаёт правила работы, а через MCP получает данные для этой работы. А самое приятное, они прекрасно работают вместе!**

💭 **От Тома:
**Скилы нужны когда у тебя есть регулярные задачи с фиксированными правилами. Ответы клиентам, посты в определённом стиле, проверка по чек-листу. Эта штука сама по себе уже неплохо работает – но интересно в ней ещё и то, что это наш маленький шаг на пути к чипам, и возможности загружать новые знания прямо в мозг 🧠
