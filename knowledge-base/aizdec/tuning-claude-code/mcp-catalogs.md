## Какие есть каталоги

Попробуем разобраться ⚔️

**🔒 Официальный реестр Anthropic** (registry.modelcontextprotocol.io)

Единственный каталог с **проверенными** серверами. Сюда попадают серверы от реальных компаний: Notion, Slack, Figma, Atlassian, Linear, HubSpot, Vercel, и других. Каждый сервер проходит верификацию. Серверов здесь сотни, не тысячи — но они работают.

_Это твоя первая остановка. Если нужный сервис есть тут — бери отсюда и не парься._

**📚 awesome-mcp-servers** на GitHub

Курируемый список MCP-серверов от сообщества. Один из самых популярных репозиториев на GitHub вообще. Серверы разбиты по категориям: базы данных, файловые системы, DevOps, коммуникации, аналитика, и так далее. Хорошая точка входа чтобы найти сервер под конкретную задачу.

_Список ведут живые люди, отбор строже чем в автоматических каталогах. Но «курируемый» не значит «безопасный» — проверять всё равно нужно._

**🔧 Smithery** (smithery.ai)

Каталог с удобной установкой. Тысячи серверов, CLI для подключения одной командой, есть возможность запускать серверы на их стороне (не у тебя на компе). Интерфейс понятный, категории, поиск.

_Удобен для быстрого подключения. Но были вопросы про безопасность CLI — код долгое время был закрыт. Для экспериментов норм, для продакшена подумай дважды._

**📊 PulseMCP** (pulsemcp.com)

Каталог + аналитика + рассылка. Показывает какие серверы набирают популярность, статистику скачиваний. Ведут email-рассылку про новинки в мире MCP.

_Полезен чтобы следить за тем, что появляется. Как каталог — средний, фильтрации по качеству нет._

**mcp.so** — самый большой по размеру

Тысячи серверов. Есть фильтр «Featured», там отобранные редакцией. Понятный интерфейс, поиск по категориям.

_Большой выбор, но раздел «Featured» это единственный способ отсеять мусор. Большинство серверов никто не проверял._

## Дикий запад каталогов 🐎 🌵

Ситуация с каталогами MCP в 2026 году — «дикий запад». Открытый стандарт без жёсткого контроля качества привёл к тому, что:

- _Многие серверы красиво описаны, но залупа на практике_

- _Все риски на пользователе, и он о них не знает :D_

- _Когда что-то ломается, трудно разобраться_

**Для нас это означает, **что вдохновляться есть чем! Но делать всё равно всё придётся самим. К счастью, не в соло — а с Claude Code.

## Важно про безопасность ☠️

MCP-сервер получает доступ к твоим данным и может выполнять действия от твоего имени. Это серьёзнее чем скиллы — потому что MCP-сервер это работающая программа, а не текстовые инструкции.

**Реальные случаи в 2026:**

- Фейковый MCP-сервер для почтового сервиса тихо копировал все отправляемые письма на адрес злоумышленника

- MCP-сервер для GitHub был атакован через «prompt injection» — злоумышленник оставил невидимые инструкции в публичном issue, и когда агент читал issue через MCP, он сливал данные из приватного репозитория

**Anthropic прямо ☠️ предупреждают:
**_«Используйте сторонние MCP-серверы на свой страх и риск»_

## Как проверить сервер перед подключением

Прежде чем подключать, попроси Claude изучить:

_Мне нужен MCP для [сервис]. Вот ссылка: [ссылка].
Изучи его: посмотри код, документацию, когда последний раз обновлялся.
Насколько он надёжный? Стоит ли подключать? Какие риски?_

**Братишка 🔎 посмотрит репозиторий,** оценит качество кода, найдёт потенциальные проблемы. И после этого (тысячу раз сомневаясь) можно переходить к установке.

## На что дыбать

**Хорошие знаки 🪴**

- Автор = известная компания или разработчик с репутацией

- Много скачиваний (тысячи)

- Свежие обновления (не старше 2-3 месяцев)

- Понятная документация

- Активные «issues» на GitHub (автор отвечает)

- Сервер есть в официальном реестре Anthropic


**Плохие знаки ☠️**

- Последнее обновление давно

- Автор непонятно кто

- В документации чёрт ногу сломит

- Скачало три человека (все боты)

- Сервер просит доступ к вещам, которые не связаны с его задачей
