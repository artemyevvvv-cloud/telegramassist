## Чем вдохновляться 👀

Экосистема скиллов для Claude Code растёт дико быстро. На GitHub десятки тысяч репозиториев, несколько каталогов, курируемые списки. Большая часть = мусор и хайп, но есть вещи на которые стоит посмотреть. Не для того чтобы слепо копировать, а чтобы понять как другие люди настраивают своего агента, и взять идеи для себя.

Ниже то, что я посчитал достойным внимания. **Ни одному из этих источников мы не доверяем на 100%**, кроме официального. Но поковырять, посмотреть, разобраться как устроено — будет полезно.

## 🔒 Официальный репозиторий Anthropic

**anthropics/skills** на GitHub — единственный источник скиллов, которому можно доверять по умолчанию. Создан и поддерживается командой Claude Code. Здесь: работа с документами (PDF, Word, Excel, презентации), инструменты разработки, и мета-скилл для создания новых скиллов.

_Немного, но надёжно. Хорошая отправная точка чтобы посмотреть как устроен «правильный» скилл — структура файла, описания, разграничение инструментов._

## 📚 Где посмотреть что вообще существует

**awesome-claude-code** на GitHub — курируемый список всей экосистемы Claude Code. Не только скиллы, но и хуки, CLAUDE.md файлы, инструменты, гайды. Обновляется часто. Лучшая точка входа если хочешь понять масштаб того, что люди делают с Claude Code.

**SkillsMP** (skillsmp.com) — веб-каталог, индексирует скиллы с GitHub. Десятки тысяч позиций, фильтры по категориям. Удобно чтобы полистать и посмотреть какие задачи люди решают скиллами. Качество разнится дико — от отличных до полностью нерабочих.

**SkillHub** (skillhub.club) — каталог поменьше, с AI-оценкой качества и песочницей для тестирования. Попытка отфильтровать мусор автоматически. AI-оценка — не замена ручной проверке, но лучше чем ничего.

_Всё это для навигации и вдохновения. Ставить оттуда что-то без проверки не стоит._

## 🔍 Что интересно изучить

**everything-claude-code** на GitHub — полная конфигурация Claude Code от победителя хакатона Anthropic. Агенты, скиллы, хуки, команды, правила, MCP-подключения — всё в одном месте. Интересно не чтобы копировать целиком, а чтобы посмотреть как выглядит продвинутая настройка. Как всё связано между собой, какие есть паттерны.

**planning-with-files** на GitHub — скилл для структурированного планирования через markdown-файлы. Решает конкретную проблему: когда задача большая и не помещается в одну сессию, скилл помогает сохранять состояние и продолжать работу. Хороший пример скилла, который решает одну задачу, но решает её хорошо.

**trailofbits/skills** на GitHub — скиллы для аудита безопасности от Trail of Bits (одна из самых известных фирм в кибербезопасности). Поиск уязвимостей, ревью кода, security-ресёрч. Один из их скиллов помог найти реальную уязвимость в криптографической библиотеке. Интересно даже если ты не занимаешься безопасностью — как пример профессионально сделанных скиллов.

_Каждый из этих репозиториев стоит открыть и покопать. Не чтобы установить всё подряд, а чтобы увидеть паттерны, структуру, подходы._

## Главная проблема: безопасность ☠️

Скилл — это не просто текст. Скилл может содержать инструкции, которые заставят Claude выполнить код на твоём компьютере.

**Что может сделать плохой скилл:**

- Прочитать твои пароли и ключи из файлов

- Отправить данные на чужой сервер

- Выполнить вредоносный код

- Скачать и запустить что-то без твоего ведома

**Правило безопасности:** относись к скиллам как к программам, которые скачиваешь из интернета — с подозрением. Звёзды на GitHub и красивый README не гарантируют безопасность.

## Как проверить перед установкой 🔦

Прежде чем ставить скилл, попроси Claude его проверить: рассмотреть содержимое на предмет подозрительных инструкций.

**На что смотреть:**

- Автор известен или аноним?

- Репозиторий активный или заброшенный?

- Есть ли понятное описание что скилл делает?

- Скилл просит доступ к чему-то лишнему? (сетевые запросы, чтение файлов за пределами проекта)

- Есть ли у других людей реальный опыт использования?

**Если по всем пунктам всё норм =** это ещё не гарантия, что всё будет збс :D

Поэтому оставайся внимателен.

💭 **От Тома:
**Для большинства задач проще попросить Claude создать скилл с нуля, чем искать, проверять и адаптировать готовый. Но насмотренность важна! Изучай что делают другие, смотри как устроены хорошие скиллы, бери идеи. Но не копируй вслепую — разберись сам, и собери то, что нужно именно тебе.
