## Ноутбук больше не нужен 📱

Весь курс ты работал с Claude Code через VS Code или терминал. Открыл ноутбук, запустил программу, сел за стол, начал работать. Удобно, привычно, понятно.

Но есть другой подход. Когда Claude Code работает на сервере — круглосуточно, без привязки к твоему компьютеру. А ты общаешься с ним **из мессенджера**. Telegram, WhatsApp, Discord — пох. Хоть с телефона в метро, хоть с планшета на пляже.

![image](https://api.aizdec.app/storage/v1/object/public/lesson-media/uploads/1772123170489-image.png)

Так сейчас работают профессионалы. И это самый логичный следующий ход, после всего, что ты узнал на моих курсах 🏴‍☠️

## Как это устроено

Да всё изи:

**[a] Сервер (VPS)** — арендуешь за $5-20/мес. На нём стоит Claude Code, твой проект, все настройки. Плюс подписка Claude, она отдельно

**[b] Мост** — программа, которая связывает мессенджер с Claude Code. Сообщение из Telegram → Claude Code → ответ обратно в Telegram

**[c] Ты** — пишешь в чат. Управляешь степень автономности агентов, и либо оцениваешь результат, либо работаешь вместе с ним

**Claude Code на сервере имеет всё то же самое, что и на ноутбуке:** файловую систему, терминал, MCP, скиллы, хуки. Просто он живёт не у тебя на столе, а в «облаке». А ты подключаешься к нему через привычный чат.

Первоначальная настройка требует технических навыков, потому что надо поставить сервер, настроить бота, подключить всё друг к другу. Но когда всё заработает, дальше будет изи.

## Какие есть подходы 🔎

### [1] «Remote Control» | официальное от Anthropic 🔒

Anthropic выпустили свою версию в конце февраля 2026. Запускаешь `claude remote-control` в терминале > появляется QR-код. Сканируешь с телефона > открывается сессия Claude Code в приложении Claude или в браузере.

**[ + ]** Плюсы: официальное, безопасное, ничего дополнительно устанавливать не нужно.

**[ - ]** Минусы: терминал на компьютере должен оставаться открытым. Закрыл крышку ноутбука — сессия умерла. Каждое действие нужно одобрять. Одна сессия на процесс.

_По сути это не «агент на сервере», а «продолжить с телефона то, что начал за компом». Полезно, но ограничено._

---

### [2] Telegram-бот на VPS | народный способ 🤖

Самый популярный паттерн в сообществе. Claude Code работает как системный сервис на VPS > стартует при загрузке, перезапускается при падении, живёт 24/7. Рядом работает Telegram-бот, который принимает твои сообщения и передаёт их Claude Code.

**Как устроен мост:**

На GitHub десятки проектов, которые это делают. Самые зрелые:

**[ + ]** Бот через SDK — полноценный контроль: сессии, расписание, вебхуки для GitHub событий

**[ + ]** Бот через tmux — мост между Telegram и терминальной сессией. Можно переключиться в терминал и увидеть всё, что происходит

**[ + ]** Минималистичные боты — 1000 строк кода, сессии в JSON, работает и не парит

**Ка решается безопасность:** бот принимает сообщения только от тебя (whitelist по Telegram ID). Некоторые проекты добавляют TOTP-коды (Google Authenticator) для одобрения опасных действий. 

---

### [3] Хуки → мессенджер | уведомления за 5 минут 🛎️

Самый простой вариант. Не управление, а **уведомления**. Настраиваешь хук (из урока 11) — и когда Claude закончил задачу, тебе приходит сообщение в Telegram.

**[ + ]** Настраивается за 5 минут, ноль зависимостей, работает с любым Claude Code.

**[ - ]** Одностороннее — ты видишь что произошло, но отправить команду обратно не можешь.

_Хорошо комбинируется с Remote Control: хук шлёт уведомление → ты открываешь Remote Control с телефона и продолжаешь._

---

### [3] OpenClaw | полноценная система 🐾

А вот это уже поинтереснее.

OpenClaw это проект, который набрал **больше 200 000 звёзд на GitHub** за пару месяцев. Это полноценный AI-ассистент: живёт на сервере, работает круглосуточно, общается через **любой мессенджер** — Telegram, WhatsApp, Discord, Signal, iMessage, Microsoft Teams.

Чем он отличается от простого Telegram-бота:

- **Мультиплатформенность** — написал в Telegram с телефона, продолжил в Discord с компьютера, агент помнит контекст

- **Память** — помнит всё, что ты ему говорил, во всех чатах, за всё время

- **Расписание** — может выполнять задачи по расписанию, без твоего участия

- **Скиллы** — расширяемая система навыков, как в Claude Code

- **Вложенные агенты** — может запускать субагентов для сложных задач

Для таких систем уже появился отдельный термин — **«Claw»**: AI-агенты, которые работают на твоём оборудовании, общаются через мессенджеры, выполняют прямые инструкции и задачи по расписанию.

Есть и более лёгкие альтернативы, например, NanoClaw: ~15 файлов исходного кода, каждая сессия в изолированном контейнере, поддержка тех же мессенджеров.

**Про OpenClaw мы поговорим подробно в следующем курсе** — там разберём установку, настройку, архитектуру, и как это использовать для реальной работы.

### Звонки по телефону. Ага, и так можно 📞

Некоторые разработчики подключают Claude Code к телефонной системе (SIP). Звонишь на номер > агент отвечает голосом > рассказывает статус задач > выполняет команды. Может сам перезвонить, если что-то случилось.

Пока это экзотика, но показывает направление. Интерфейс может быть **любым**, главное, что агент работает и доступен.

## Что это меняет (всё)

Вот эволюция того, как мы взаимодействовали с кодом:

```text
Терминал (SSH) → IDE (VS Code) → Мессенджер
```

На каждом шаге снижается порог входа и растёт доступность. Терминал требует знания команд. IDE установки и настройки. А мессенджер нихера не требует, кроме умения формулировать свои мысли. 

**Правда.. Это не полная замена терминала и IDE для некоторых из нас.**

Для глубокой работы с кодом, сложных задач — VS Code и терминал никуда не денутся, они намбер ван. Но далеко не все задачи относятся к этой категории. Для 90% из них – мессенджера достаточно.

Один из создателей Claude Code теперь пишет треть своего кода с iOS-приложения. Люди пачками запускают агентов, которые работают неделями автономно — и сами принимают решения, деплоят код. Проверяют всё только с телефона. Это тренд, к которому уже пора приготовиться 🥷🏽

А если вспомнить предыдущий урок, то можно сообразить, что и Agent Teams тоже можно развернуть на VPS и управлять ими с телефона. Лид-агент будет координировать, тиммейты работать, а ты контролировать ситуацию в телеграчике.

Если агенты правильно организованы — с чёткими скиллами, хуками, правилами, контрактами (всё то, чему ты учился в этом курсе), то тебе достаточно давать им **направление**. И они сами будут охуенно делать, пиздато проверять, и чётко координироваться ⚔️

## Что дальше 😈

Ты научился работать с агентом: давать задачи, строить скиллы, подключать MCP, настраивать хуки, работать с параллельными сессиями, координировать команды агентов.

Следующий шаг >>> **поставить всё это на сервер и управлять с любого устройства**. Не только для разработки. Агенты умеют решать задачи от технических до маркетинговых, от юридических до инвестиционных. Граница только в том, насколько хорошо ты умеешь ставить задачи и контролировать результат.
